# 📐 มาตรฐานบทเรียน (บังคับใช้ทุกเนื้อหาที่สร้างในคอร์สนี้)
> กำหนดโดยวี · มิ.ย. 2026 — Claude ต้องตรวจ checklist นี้ก่อนส่งบทเรียนทุกครั้ง

ทุกบทเรียนต้องมีครบ 6 ส่วน ตามลำดับ:

## 1. 🧠 แนวคิด & หลักการของ Dev (Concept First)
ก่อนเข้า syntax ต้องตอบ 3 คำถามนี้เสมอ:
- **ปัญหาอะไร** ที่ทำให้สิ่งนี้ถูกสร้างขึ้นมา (ทุกเครื่องมือเกิดจากความเจ็บปวดบางอย่าง)
- **หลักคิด (principle)** ที่อยู่เบื้องหลัง เช่น "Don't block the main thread", "Fail fast", "Separation of concerns"
- **เมื่อไหร่ใช้ / เมื่อไหร่ไม่ใช้** — เพราะรู้แค่ "ใช้ยังไง" โดยไม่รู้ "เมื่อไหร่" = เขียนเองในอนาคตไม่ได้

## 2. 🔍 ทำไมต้องใช้ code นี้ (Why ทีละบรรทัด)
- โค้ดตัวอย่างสำคัญทุกชิ้น ต้องมีคำอธิบาย "ทำไม" ไม่ใช่แค่ "ทำอะไร"
  - ❌ "ใส่ await หน้า fetch" (บอกแค่ what)
  - ✅ "ใส่ await เพราะ fetch คืน Promise — ถ้าไม่รอ ตัวแปรจะได้กล่องเปล่าที่ยังไม่มีของ (Promise pending) ไม่ใช่ข้อมูล" (บอก why + ผลถ้าไม่ทำ)
- แสดง "ถ้าไม่ใช้จะเกิดอะไร" (counter-example) อย่างน้อย 1 จุดต่อแนวคิดหลัก

## 3. 🌏 ตัวอย่างการนำไปใช้งานจริง
อย่างน้อย 2 บริบทต่อบท:
- **บริบทงานของวี:** media-presto (WordPress, GA4/Meta automation, GEO/AEO) หรือโปรเจกต์ที่มีจริง (welovefukushima, library-system)
- **บริบทอุตสาหกรรม:** ระบบที่คนรู้จัก (e-commerce, banking app, social media) เพื่อเห็นภาพว่าของจริงระดับโลกใช้ยังไง

## 4. 🛠 Mission ที่จบเป็นผลงานได้
- ทุก Mission ใหญ่ต้องจบเป็นของที่ **push ขึ้น Git ได้อย่างภาคภูมิ** ตามมาตรฐานใน `portfolio-standard.md`
- มีเกณฑ์เทสต์ชัดเจนทุกขั้น (ผ่าน/ไม่ผ่าน ตัดสินได้เอง)

## 5. 🖼 ภาพประกอบ + คีย์เวิร์ดภาพจำ (Visual First)
- แนวคิดหลักของบท ต้องมี **diagram อย่างน้อย 1 ภาพ** (SVG ฝังในไฟล์ — flow, timeline, เปรียบเทียบ ก่อน/หลัง)
- ทุกแนวคิดสำคัญต้องมี **คีย์เวิร์ดภาพจำ (mnemonic keyword)** — คำสั้น ๆ ที่เรียกภาพในหัวกลับมาได้
  เช่น async = "บาริสต้าคนเดียว", Promise = "ใบรับของ 3 สถานะ", fetch = "พนักงานส่งของ 2 จังหวะ", localStorage = "ตู้เก็บของที่ใคร ๆ ในห้องก็เปิดได้"
- แสดงคีย์เวิร์ดเป็น chip/ป้ายชัดเจนต้นบท เพื่อใช้ตอน Friday Rebuild (เห็นคีย์เวิร์ด → ดึงภาพ → ดึงโค้ดได้)
- หลักการ: ถ้าอธิบาย 3 ย่อหน้าแล้วยังต้องอ่านซ้ำ = ควรเป็นภาพ

## 6. 📝 Quiz ที่ทดสอบ "ความเข้าใจ" ไม่ใช่ "ความจำ"
- ถามแบบ "ทำไม / จะเกิดอะไรถ้า / เลือกท่าไหนเพราะอะไร" ไม่ถามแบบท่องจำ syntax
- อย่างน้อย 1 ข้อต้องเชื่อมกับ security หรือ production จริง

## Checklist ก่อนส่งบทเรียน (Claude ใช้ตรวจตัวเอง)

- [ ] โค้ดหลักทุกชิ้นมี why + counter-example
- [ ] ตัวอย่างใช้จริง ≥ 2 บริบท (งานวี + อุตสาหกรรม)
- [ ] Mission จบเป็นผลงาน push ได้ + เกณฑ์เทสต์ครบ
- [ ] **diagram ≥ 1 ภาพ + คีย์เวิร์ดภาพจำครบทุกแนวคิดหลัก**
- [ ] Quiz เน้นความเข้าใจ + มีข้อ security/production
