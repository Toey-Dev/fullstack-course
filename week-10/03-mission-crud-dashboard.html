Week 10 · Mission ใหญ่

📋 Books/Users CRUD + Dashboard

เติมเนื้อให้ library-web เป็นแอป CRUD เต็มระบบ — ปิด Kcode Day 8 สมบูรณ์

ทำไม Mission นี้ทำให้แอป "ใช้งานได้จริงครบ flow"

หลังจบ library-web จะจัดการหนังสือและผู้ใช้ได้ครบ (เพิ่ม/แก้/ลบ), ยืม-คืนผ่าน UI, มี dashboard ภาพรวม — เป็นแอปที่ demo ให้ใครดูก็ "เสร็จจริง" · นี่คือทักษะ CRUD + form ที่ใช้ทำ business app อะไรก็ได้

ต่อยอด library-web จาก Week 9 (มี app shell + auth แล้ว) + backend รันอยู่

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 10.1 — Books CRUD 90 นาที

  1. /books — list (4 สถานะ) + ปุ่ม add/edit/delete
  2. BookForm เดียวใช้ทั้ง add (/books/add) และ edit (/books/[id]/edit prefill)
  3. delete + confirm + refetch · validate client + backend (Week 4 Zod)
✅ เทสต์: เพิ่มหนังสือ → เห็นใน list ทันที · แก้ → ค่า prefill ถูก · ลบ → confirm + หาย

Mission 10.2 — Users CRUD (admin only) 60 นาที

  1. /users — list ผู้ใช้ (เฉพาะ admin เห็น — กัน 3 ชั้น จาก Week 9)
  2. add/edit user + จัดการ role
  3. ไม่แสดง password (Week 5) · backend ตรวจ role อีกชั้น
security: user ทั่วไปเข้า /users → 403/redirect · backend requireRole("admin") ทุก endpoint

Mission 10.3 — Borrow/Return UI 60 นาที

  1. ปุ่มยืมในหน้า book detail → เรียก /api/borrow (transaction จาก Week 5)
  2. /borrows — รายการที่ยืม + ปุ่มคืน
  3. อัปเดต UI หลังยืม/คืน (refetch) · จัดการ error (ยืมซ้ำ → แสดงข้อความ)
✅ เทสต์: ยืม → หนังสือเปลี่ยนสถานะ · ยืมเล่มที่ถูกยืมแล้ว → error · คืน → กลับว่าง

Mission 10.4 — Dashboard + push 60 นาที

  1. backend: aggregate endpoint (COUNT หนังสือ/ยืม/สมาชิก)
  2. /dashboard — stat cards (Promise.all) + responsive grid
  3. push — README screenshot ทุกหน้า + AI Usage
✅ เทสต์: dashboard แสดงตัวเลขจริงจาก DB · ตรงกับข้อมูลในหน้า books/users
