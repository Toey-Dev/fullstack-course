🧹 Clean Code ประจำ Week 6

1. ปล่อยให้ TS infer — อย่า annotate เกินจำเป็น

❌ ใส่ type ทุกที่ (รก)

const name: string = "Vee";
const nums: number[] = [1, 2, 3];
const double = (n: number): number => n * 2;

✅ ใส่เฉพาะที่ TS เดาเองไม่ได้

const name = "Vee";        // TS รู้ว่า string
const nums = [1, 2, 3];    // TS รู้ว่า number[]
const double = (n: number) => n * 2;  // param ต้องบอก, return infer เอง

2. หลีกเลี่ยง any — มันคือ "ปิดสวิตช์ TS"

❌ any = กลับไปเป็น JS

function process(data: any) {
  return data.titel;  // พิมพ์ผิด TS ไม่เตือน!
}

✅ type จริง หรือ unknown

function process(data: Book) {
  return data.titel;  // ❌ TS เตือนทันที: ไม่มี titel
}
กฎทอง: ใส่ type ที่ "ขอบ" (input ฟังก์ชัน, req.body, return ของ API) ปล่อย infer ข้างใน · เลี่ยง any (ใช้ unknown ถ้าไม่รู้จริง) · 1 type ใช้ซ้ำ (สร้าง Book type ที่เดียว) · เปิด strict mode
🤖 AI Code Review Drill #5 (ศุกร์ 30 นาที): ขอให้ AI แปลงฟังก์ชัน JS เป็น TS แล้วหาจุดที่ AI ใช้ any เลี่ยงปัญหา (≥2 จุด) — AI ชอบใส่ any เมื่อ type ซับซ้อน ซึ่งทำให้เสียประโยชน์ TS ทั้งหมด · จดลง notes/ai-review-log.md
