🔷 แปลง library-system เป็น TypeScript

ผ่าตัดทั้งระบบเป็น TS strict — โดยมี test เป็นตาข่ายว่าแปลงแล้วยังทำงานถูก

ทำไม Mission นี้คือก้าวเปลี่ยนระดับ

หลังจบ Mission นี้ library-system เป็น TypeScript strict ทั้งระบบ — resume คุณเพิ่ม "TypeScript" ได้อย่างมีหลักฐานจริง และระหว่างแปลง คุณจะเจอบั๊กที่ JS ซ่อนไว้ (ค่าที่อาจ undefined, type ที่ส่งผิด) ที่ strict mode จับให้ — test (Week 3) ยืนยันว่าหลังแปลงระบบยังทำงานถูก · นี่คือการ refactor ใหญ่ที่ปลอดภัยเพราะมี test

กลยุทธ์: แปลงทีละชั้น (ล่างขึ้นบน) ไม่ใช่ทีเดียวทั้งระบบ — แปลงชั้นล่าง รัน build ดู error แก้ แล้วค่อยขึ้นชั้นถัดไป

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 6.1 — Setup TS + types พื้นฐาน 45 นาที

  1. ติดตั้ง typescript, tsx, @types/node, @types/express + tsc --init
  2. tsconfig: "strict": true, rootDir src, outDir dist
  3. scripts: dev (tsx watch), build (tsc), start (node dist) — ปรับ test ให้รัน TS
  4. สร้าง src/types/book.ts — interface Book หรือ z.infer จาก BookSchema
✅ เทสต์: npm run build ผ่าน (ยังไม่มี error) · npm run dev server รัน

Mission 6.2 — แปลงชั้น data + repository 60 นาที

  1. เปลี่ยน .js → .ts ของ data/repository
  2. type ของ repository function: findById(id: number): Promise<Book | null>
  3. type ผลจาก pg query (rows เป็น Book[])
จุดที่ strict จะเตือน: rows[0] อาจ undefined → ต้องจัดการ (คืน null) — นี่คือบั๊กที่ JS ซ่อนไว้

Mission 6.3 — แปลง controller + middleware + route 75 นาที

  1. controller: type (req: Request, res: Response) + type req.body จาก Zod infer
  2. auth middleware (Week 5): type req.user (ขยาย Request type)
  3. error handler: type (err: Error, req, res, next)
✅ เทสต์: npm run build ผ่านโดยไม่มี any (ยกเว้นจุดที่จำเป็นจริง + comment เหตุผล)

Mission 6.4 — test ยังเขียว + push 60 นาที

  1. ปรับ test เป็น TS (หรือให้ vitest รัน .ts) — npm test ต้องเขียว
  2. ไล่ลบ any ที่หลงเหลือ (เป้า: 0 any ที่ไม่จำเป็น)
  3. push — README อัปเดต Tech Stack เพิ่ม TypeScript + AI Usage
✅ เทสต์: build ผ่าน + test เขียว + ไม่มี any ลอย ๆ = แปลงสำเร็จสมบูรณ์
