🗄 library-system → PostgreSQL + Zod

ย้ายจากไฟล์ JSON สู่ database จริง — โดยมี test จาก Week 3 เป็นตาข่ายรองรับให้กล้าแก้

ทำไม Mission นี้คือบทพิสูจน์ของ Week 3

คุณกำลังจะ "ผ่าตัดใหญ่" — เปลี่ยนที่เก็บข้อมูลทั้งระบบ ปกติน่ากลัวมาก (กลัวพังที่อื่น) แต่คุณมี test จาก Week 3 เป็นตาข่าย: แก้เสร็จรัน npm test ถ้ายังเขียว = ผ่าตัดสำเร็จ นี่คือเหตุผลจริงว่าทำไมเราเขียน test ก่อน — เพื่อให้สัปดาห์นี้กล้าแก้

เรียนคู่ Kcode Day 4 (Connect Express to PostgreSQL) ได้ — เขียนเองทุกบรรทัด ปิด AI

📋 เป้าหมาย

library-system เดิม (เก็บ books ใน array/JSON) → เก็บใน PostgreSQL ที่รันใน Docker · validate ด้วย Zod · test เดิมยังเขียว

โครงที่จะมีหลังจบ:

library-system/
├── docker-compose.yml          # PostgreSQL container
├── .env / .env.example         # DATABASE_URL (ไม่ commit .env)
├── src/
│   ├── db.js                   # connection pool
│   ├── schemas/book.schema.js  # Zod
│   ├── repositories/books.repo.js  # คุย DB (SQL)
│   ├── controllers/ routes/ middlewares/
│   └── app.js / server.js
└── tests/

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 4.1 — Docker + ตาราง 45 นาที

  1. เขียน docker-compose.yml รัน PostgreSQL + docker compose up -d
  2. ต่อ DBeaver + รัน CREATE TABLE books (id, title, author, available, created_at)
  3. ตั้ง .env (DATABASE_URL) + ใส่ .env, node_modules ใน .gitignore
✅ เทสต์: DBeaver เห็นตาราง books · git status ไม่มี .env หลุด

Mission 4.2 — Repository layer + Zod 60 นาที

  1. src/db.js — connection Pool
  2. src/repositories/books.repo.js — findAll, findById (ใช้ $1 parameterized)
  3. src/schemas/book.schema.js — Zod BookSchema + validate middleware
คิดแบบ pentester: ทุก query ที่มี input ต้องใช้ $1 — ไม่มีที่ไหนต่อ string เด็ดขาด

Mission 4.3 — CRUD เชื่อม DB จริง 75 นาที

  1. controller เรียก repository (ไม่แตะ SQL ตรง ๆ ใน controller)
  2. GET /api/books + GET /api/books/:id (404 ถ้าไม่เจอ) — อ่านจาก DB จริง
  3. POST /api/books ผ่าน Zod validate → INSERT → คืน 201 + book ที่สร้าง
✅ เทสต์มือ: POST เล่มใหม่ → เห็นใน DBeaver จริง · restart server → ข้อมูลยังอยู่ (ต่างจาก JSON เดิม)

Mission 4.4 — test ยังเขียว + push 60 นาที

  1. ปรับ test (Week 3) ให้รันกับ DB — ตั้ง test data ก่อนเทสต์, เคลียร์หลังเทสต์ (หรือใช้ test DB แยก)
  2. npm test ต้องเขียว — พิสูจน์ว่าผ่าตัดสำเร็จ ระบบยังทำงานถูกหลังเปลี่ยน DB
  3. push — README เพิ่มหัวข้อ "Setup" (docker compose up + .env.example) + AI Usage
✅ เทสต์: clone repo ใหม่ → ตาม README → ระบบรันได้ (นี่คือบททดสอบจริงว่า README ดีพอ)
