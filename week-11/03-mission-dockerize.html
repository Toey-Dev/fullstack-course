Week 11 · Mission ใหญ่

🐳 Dockerize library-system เต็มระบบ

แพ็คทั้งระบบลง container — clone แล้ว docker compose up ได้เลย

ทำไม Mission นี้คือทักษะ DevOps ชิ้นแรก

หลังจบ library-system จะ "รันได้ทุกที่ด้วยคำสั่งเดียว" — Dockerfile multi-stage (image เล็ก), compose app+db+redis, cache ด้วย Redis · นี่คือสิ่งที่ทำให้ deploy ขึ้น cloud ได้ (Week 14) และเป็นทักษะที่ JD DevOps/Backend ขอ · มุม pentester: image ที่ปลอดภัย (ไม่มี secret, ไม่รัน root) คือจุดเด่นของคุณ

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 11.1 — Dockerfile multi-stage 75 นาที

  1. เขียน Dockerfile multi-stage (build → run slim) สำหรับ library-system (TS → dist)
  2. เรียง copy package + npm ci ก่อน copy source (layer caching)
  3. .dockerignore (node_modules, .env, .git, dist)
  4. build + เทียบขนาด image (single vs multi-stage)
✅ เทสต์: docker build ผ่าน · image multi-stage เล็กกว่า single ชัดเจน

Mission 11.2 — Compose app+db+redis 60 นาที

  1. compose 3 service: app (build .), db (postgres), redis
  2. app เชื่อม DB ด้วยชื่อ service (db:5432 ไม่ใช่ localhost) + redis
  3. depends_on + healthcheck (รอ db พร้อมจริง) · volume db
✅ เทสต์: docker compose up → ทั้ง 3 service รัน · API ตอบได้ · ข้อมูลรอด restart

Mission 11.3 — Redis cache 60 นาที

  1. cache endpoint ที่เรียกบ่อย (GET /api/books หรือ /api/dashboard) ด้วย Redis (TTL 60 วิ)
  2. invalidation: ลบ cache เมื่อ POST/PUT/DELETE books
  3. ทดสอบ: ดู log/เวลา — ครั้งแรก query DB, ครั้งถัดไปจาก cache
คิดให้รอบ: เพิ่มหนังสือแล้วต้องโผล่ทันที (invalidate ถูก) — ไม่ใช่รอ cache หมดอายุ

Mission 11.4 — Clone test + push 45 นาที

  1. ทดสอบจริง: clone repo ไปโฟลเดอร์ใหม่ → docker compose up → ใช้งานได้ (บททดสอบว่า setup ครบ)
  2. README: วิธีรัน (compose up อย่างเดียว) + .env.example + Tech Stack เพิ่ม Docker/Redis
  3. push + AI Usage
✅ เทสต์: clone ใหม่ → compose up → ระบบรันครบโดยไม่ต้องลงอะไรเพิ่ม = setup สมบูรณ์
