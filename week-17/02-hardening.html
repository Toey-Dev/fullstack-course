Hardening — Helmet, Rate Limit, CSP, CORS

เครื่องมือป้องกันที่ใส่เป็น middleware — เกราะที่ทุก production ต้องมี

🔑 คีย์เวิร์ดภาพจำประจำบท

🪖 หมวกกันน็อค headers = helmet 🚦 จำกัดจำนวนยิง = rate limit 📜 อนุญาตแหล่ง script = CSP 🌐 ใครเรียกข้าม origin ได้ = CORS

🧠 แนวคิด & หลักการของ Dev ประจำบทนี้

หลักการ 1: "Security headers — บอก browser ให้ช่วยป้องกัน"

browser มีกลไกป้องกันในตัว (กัน clickjacking, XSS, MIME sniffing) แต่ต้อง "เปิด" ด้วย HTTP headers · helmet ตั้ง headers เหล่านี้ให้ · นี่คือ "ให้ browser เป็นแนวร่วมป้องกัน" — header เดียวกันกัน attack หลายแบบ ฟรี ๆ

หลักการ 2: "Rate limit — abuse ไม่ใช่แค่ traffic เยอะ"

ไม่จำกัดจำนวน request → brute force password (ลองพันรหัส/วินาที), scrape ข้อมูล, DoS (ยิงจนล่ม) · rate limit จำกัด "คนเดียวยิงได้กี่ครั้ง/ช่วงเวลา" · โดยเฉพาะ /login (กัน brute force) · คุณเคยใช้ brute force เจาะ — รู้ว่า rate limit สำคัญแค่ไหน

หลักการ 3: "CORS — กันเว็บอื่นเรียก API เราในนามผู้ใช้"

browser ห้าม JS จากเว็บ A เรียก API เว็บ B (same-origin policy) เว้นแต่ B อนุญาต (CORS) · ตั้ง CORS ผิด (allow *) = เว็บร้ายเรียก API เราด้วย cookie ผู้ใช้ได้ · allow เฉพาะ origin ที่รู้จัก (frontend ของเราเท่านั้น)

ส่วนที่ 1: Helmet — Security Headers

npm install helmet
app.use(helmet());          // ใส่ security headers ทั้งชุด (1 บรรทัด)
header (helmet ตั้งให้)กันอะไร
Content-Security-PolicyXSS (จำกัดแหล่ง script — ส่วนที่ 3)
X-Frame-Options: DENYClickjacking (กันเว็บอื่นเอาเราใส่ iframe หลอกคลิก)
Strict-Transport-Security (HSTS)บังคับ HTTPS (กัน downgrade เป็น HTTP)
X-Content-Type-Options: nosniffกัน MIME sniffing (browser เดา type ผิดแล้วรัน)
ซ่อน X-Powered-Byไม่บอกว่าใช้ Express (ลด recon — มุม pentester)

helmet 1 บรรทัดตั้ง header ป้องกันหลายแบบ · ปรับแต่ง CSP เพิ่มได้ (ส่วนที่ 3) · นี่คือ low-effort high-impact ที่ทุก production ต้องมี

ส่วนที่ 2: Rate Limiting

npm install express-rate-limit

import rateLimit from "express-rate-limit";

// ทั่วไป — 100 req/15 นาที/IP
app.use(rateLimit({ windowMs: 15*60*1000, max: 100 }));

// /login เข้มกว่า — กัน brute force (5 ครั้ง/15 นาที)
const loginLimiter = rateLimit({
  windowMs: 15*60*1000, max: 5,
  message: { ok: false, error: "พยายาม login มากเกินไป ลองใหม่ภายหลัง" },
});
app.post("/api/login", loginLimiter, loginController);
มุม pentester ของคุณ: คุณเคย brute force /login มาแล้ว — รู้ว่าถ้าไม่มี rate limit ลองรหัสได้ไม่จำกัด · /login, /register, /reset-password ต้อง rate limit เข้มเป็นพิเศษ · production จริงเก็บ counter ใน Redis (W11) เพื่อให้ข้าม instance ได้ (rate limit ใน memory ไม่ work ถ้ามีหลาย server)

ส่วนที่ 3: Content Security Policy (CSP)

// CSP — บอก browser ว่า script/style/img โหลดจากไหนได้บ้าง
app.use(helmet.contentSecurityPolicy({
  directives: {
    defaultSrc: ["'self'"],                    // default โหลดจากเว็บตัวเองเท่านั้น
    scriptSrc: ["'self'"],                     // script จากเราเท่านั้น
    imgSrc: ["'self'", "https://cdn.example.com"],
    objectSrc: ["'none'"],                     // ไม่อนุญาต
