คุณเคยเจาะระบบตาม OWASP Top 10 มาแล้วในกองทัพ — Week 17 พลิกมุม: ทำให้ library-system ทนต่อ OWASP Top 10 · คุณได้เปรียบ dev ทั่วไปมหาศาล เพราะ "รู้ว่าผู้โจมตีคิดยังไง" → ป้องกันได้ตรงจุด · จบสัปดาห์นี้ระบบจะ hardened ครบทุกชั้น และคุณจะเล่า interview สาย security ได้ว่า "เข้าใจทั้งฝั่งโจมตีและป้องกัน"

  • OWASP Top 10 (2021) ฝั่งป้องกัน — ทบของที่เรียนแล้ว + เติมที่ขาด
  • Security headers ด้วย helmet (CSP, HSTS, X-Frame-Options ฯลฯ)
  • Rate limiting — กัน brute force, DoS, abuse
  • Input sanitization + output encoding (กัน XSS, injection)
  • CORS ที่ถูกต้อง + security misconfiguration
ทำไมเฟสนี้คือ moat ของคุณ: dev ที่ "เขียน secure code เป็น + เคย pentest จริง" หายากมากในไทย · นี่คือสิ่งที่ทำให้คุณสมัคร Product Security/AppSec ได้ (สาย B เงินเดือนสูง) และโดดเด่นแม้ในตำแหน่ง dev ทั่วไป

🧹 Clean Code ประจำ Week 17

1. Security เป็น middleware กลาง ไม่กระจาย

❌ เช็ค security ทีละ route

app.get("/a", (req,res)=>{ /* เช็คเอง */ });
app.get("/b", (req,res)=>{ /* เช็คเอง ลืมบ้าง */ });

✅ middleware ครอบทั้งแอป

app.use(helmet());           // headers ทุก response
app.use(rateLimit({...}));    // จำกัดทุก request
app.use(cors({...}));         // CORS policy เดียว
// ครอบทั้งระบบ ลืมไม่ได้

2. Secure by default — default คือ "ปิด" แล้วเปิดเท่าที่จำเป็น

CORS allow เฉพาะ origin ที่รู้จัก (ไม่ใช่ *) · rate limit ทุก endpoint · CSP จำกัด source · default ปลอดภัยก่อน ค่อยผ่อน

กฎทอง: security เป็น middleware กลาง · secure by default (ปิดก่อน เปิดเท่าจำเป็น) · ป้องกันหลายชั้น (defense in depth) · validate input + encode output · ไม่เชื่อ input ทุกแหล่ง
