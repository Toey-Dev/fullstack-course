Week 1 — ปิดจบ Frontend ให้สนิท

Async/Await · Fetch API · LocalStorage · Mission ใหญ่ "My Reading List" · Deploy จริง

🎯 เป้าหมายของสัปดาห์นี้

สัปดาห์นี้คือ "ด่านสุดท้ายของ Frontend" ก่อนเข้าสู่ Backend เต็มตัว เราจะเก็บ 2 เรื่องที่เหลือจากคอร์ส Zero to One (Week 5) ให้แน่นถึงแก่น แล้วพิสูจน์ด้วยชิ้นงานจริง 1 ชิ้นที่ deploy ขึ้นอินเทอร์เน็ตได้

  • เข้าใจ Async/Await ลึกพอที่จะ "อธิบายให้คนอื่นฟังได้" ไม่ใช่แค่ใช้เป็น
  • ใช้ Fetch + LocalStorage แบบมี error handling ครบเหมือนงาน production
  • สร้างแอป "My Reading List" เสร็จ + ขึ้น GitHub Pages

บทเรียน

บทที่ 1 Async/Await ให้เข้าใจถึงแก่น

ทำไม JavaScript ต้องมีระบบ "ไม่รอคิว" · วิวัฒนาการ Callback → Promise → Async/Await · Mission เครื่องชงกาแฟ · โจทย์ทายลำดับ output แบบที่เจอใน interview จริง

⏱ ~1.5 ชม. (อ่าน 30 นาที + Mission 1 ชม.)

บทที่ 2 Fetch API + LocalStorage (มุมมอง Pentester)

ดึงข้อมูลจาก API จริงแบบครบวงจร loading/error · กฎเหล็กของ LocalStorage · อะไรห้ามเก็บและทำไม (XSS) · Mission Pokedex มินิ + ระบบ Favorites

⏱ ~2 ชม. (อ่าน 30 นาที + Mission 1.5 ชม.)

Mission ใหญ่ My Reading List — ชิ้นงานปิดจบ Frontend

รวมทุกทักษะเป็นแอปจริง 1 ชิ้น แบ่งเป็น 5 Mission ย่อยพร้อมวิธีเทสต์ทุกขั้น จบด้วยการ deploy ขึ้น GitHub Pages ได้ URL ผลงานออนไลน์ชิ้นแรก

⏱ ~4 ชม. (แบ่งทำ 2-3 วันได้)
