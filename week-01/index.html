🧹 Clean Code ประจำ Week 1 — รูปแบบ async/fetch ที่ dev จริงใช้

1. async/await ชนะ .then() chain (อ่านเหมือนโค้ดปกติ)

❌ .then ซ้อน อ่านยาก

fetch(url)
  .then(res => res.json())
  .then(data => {
    return fetch(url2);
  })
  .then(res => res.json())
  .then(d => console.log(d))
  .catch(e => ...);

✅ async/await เรียงตรง

try {
  const res = await fetch(url);
  const data = await res.json();
  const res2 = await fetch(url2);
  const d = await res2.json();
  console.log(d);
} catch (e) { ... }

2. Guard clause — เช็คแล้ว return เร็ว (เลี่ยง if ซ้อน)

❌ ซ้อนลึก (pyramid)

if (res.ok) {
  const data = await res.json();
  if (data.items) {
    render(data.items);
  }
}

✅ ดักออกก่อน เนื้อหลักไม่ย่อหน้าลึก

if (!res.ok) throw new Error(res.status);
const data = await res.json();
if (!data.items) return;
render(data.items);

3. Optional chaining (?.) + Nullish (??) — กัน null สวย ๆ

❌ เช็คมือทีละชั้น

const name = data &&
  data.user &&
  data.user.name
    ? data.user.name : "ไม่ระบุ";

let saved = localStorage.getItem("x");
if (saved === null) saved = "[]";

✅ สั้นและปลอดภัย

const name = data?.user?.name ?? "ไม่ระบุ";



const saved = localStorage.getItem("x") ?? "[]";
เอาไปใช้ใน Mission เลย: Pokedex และ Reading List ใช้ทั้ง 3 รูปแบบนี้ได้ครบ — เขียนแบบ ✅ ตั้งแต่แรก จะติดนิสัยและ reviewer (ผม) ให้คะแนนส่วน "คุณภาพโค้ด" สูงขึ้น
กติกาสำคัญ: Mission ทุกข้อเขียนเองทั้งหมด ห้ามเปิดเฉลยก่อนพยายามครบ 20 นาที — ความรู้สึก "ติดขัด" คือตอนที่สมองกำลังสร้างความจำระยะยาว ซึ่งเป็นสิ่งที่คุณบอกว่าขาดมาตลอด
