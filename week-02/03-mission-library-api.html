📚 library-system v0.1 — Books API

API จริงชิ้นแรกของโปรเจกต์ที่จะอยู่กับคุณทั้งคอร์ส — เริ่มต้นให้ถูก เพราะอีก 22 สัปดาห์ต่อยอดจากนี้

ทำไม Mission นี้สำคัญเป็นพิเศษ

นี่ไม่ใช่แบบฝึกหัดทิ้ง — library-system คือ ยานพาหนะหลักของทั้งคอร์ส สิ่งที่สร้างวันนี้จะถูกเติม TypeScript (W6), test (W3), database (W4), auth (W5), Docker (W11), CI/CD (W13) และถูกคุณ pentest เอง (W20) จนกลายเป็น portfolio piece ที่เล่าใน interview ได้ 30 นาที โครงที่วางวันนี้ = ฐานรากของบ้านทั้งหลัง

เรียนคู่ Kcode Day 2 (Books CRUD) ได้ — แต่ เขียนเองทุกบรรทัด ปิด AI ดูคอร์สเป็นไอเดีย ไม่ copy

📋 สเปค (แบบที่ลูกค้า/ทีมจะสั่งจริง)

Books API — จัดการรายการหนังสือในห้องสมุด:

Method + Pathหน้าที่ตอบกลับ
GET /api/healthเช็คชีพจร (จาก W2 บท 1)200 + {ok, time}
GET /api/booksดูหนังสือทั้งหมด + รองรับ ?search=200 + รายการ
GET /api/books/:idดูเล่มเดียว200 + เล่ม / 404 ถ้าไม่เจอ
POST /api/booksเพิ่มเล่มใหม่ (มี validation)201 + เล่มใหม่ / 400 ถ้า input ผิด

ข้อมูลหนังสือ: { id, title, author, available } · เก็บลงไฟล์ JSON ให้รอด restart

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 3.1 — โครงสร้าง + GET ทั้งหมด 60 นาที

  1. วางโครง src/ (server, routes/, controllers/, data/) ตามบทที่ 2
  2. data/books.js — array หนังสือเริ่มต้น 3-4 เล่ม
  3. GET /api/books คืนทั้งหมด แยก logic ไป controller (server.js ต้องบาง)
✅ เทสต์: localhost:3000/api/books เห็น array · server.js มีแค่ setup + ผูก route (ไม่มี logic ปน)

Mission 3.2 — GET by id + query search 45 นาที

  1. GET /api/books/:id — เจอคืน 200, ไม่เจอคืน 404 + error message (อย่าลืมเคสไม่เจอ)
  2. GET /api/books?search=คำ — กรอง title ที่มีคำนั้น (ใช้ .filter() + .includes() จาก Week 1)
✅ เทสต์: ขอ id ที่ไม่มี ต้องได้ 404 ไม่ใช่ 200 + null / search แล้วได้เฉพาะที่ตรง

Mission 3.3 — POST + validation + persistence 75 นาที

  1. POST /api/books — validate title (จำเป็น, เป็น string) ผิด = 400, ผ่าน = 201
  2. บันทึกลงไฟล์ JSON จริง (fs module) — เพิ่มเล่มแล้ว restart server ต้องยังอยู่
  3. error message ปลอดภัย (ไม่รั่วโครงสร้างภายใน — มุม pentester)
✅ เทสต์: POST เล่มใหม่ → restart → GET เห็นเล่มนั้นอยู่ / POST body ว่าง → ได้ 400

Mission 3.4 — ขึ้น GitHub ตามมาตรฐาน 40 นาที

  1. .gitignore ใส่ node_modules (สำคัญ! ห้าม commit)
  2. README ตาม portfolio-standard.md: Features, Tech Stack, Setup, What I Learned, AI Usage ("No AI in core logic")
  3. commit แบบมีจังหวะ: feat: add books list endpoint ... ไม่ใช่ก้อนเดียว
  4. push repo library-system (public ได้ — ยังไม่มีความลับ)
✅ เทสต์: เปิด repo บน GitHub — README สวย, ไม่มี node_modules, commit history เล่าเรื่องได้
