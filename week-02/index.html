Week 2 — Node.js + Express: กำเนิด library-system

จาก Browser สู่ Server · สร้าง API แรกของโปรเจกต์ที่จะอยู่กับเราทั้งคอร์ส

🎯 เป้าหมายของสัปดาห์นี้

สัปดาห์ที่แล้ว JavaScript ของคุณวิ่งใน browser — สัปดาห์นี้มันจะวิ่งบน server เราจะเริ่มโปรเจกต์ library-system (ระบบห้องสมุดจากคอร์ส Kcode) ซึ่งจะถูกต่อยอดไปจนจบคอร์ส: สัปดาห์นี้เป็น Express API ธรรมดา → อีก 22 สัปดาห์ข้างหน้ามันจะมี TypeScript, test, Docker, CI/CD และผ่านการ pentest โดยคุณเอง

  • เข้าใจว่า Node.js คืออะไร ต่างจาก JS ใน browser ยังไง และทำไม single thread ถึงรับพันคนได้
  • สร้าง Express server + endpoint แรก พร้อมเข้าใจทุกบรรทัดว่า "ทำไม"
  • จัดโครงโปรเจกต์แบบมืออาชีพ (routes / controllers / data) ตั้งแต่วันแรก
  • library-system ขึ้น GitHub เป็น repo จริงตาม portfolio-standard

เรียนคู่กับ Kcode Day 1-2 (docs.kcode.online) — บทเรียนนี้คือ "เลนส์" ช่วยให้อ่านคอร์สแล้วเข้าใจลึกขึ้น ไม่ใช่ของทดแทน

🧹 Clean Code ประจำ Week 2 — รูปแบบ Express ที่ใช้ในทีมจริง

1. Destructure req ทันที (ไม่เขียน req.body.x ซ้ำ ๆ)

❌ req.body ทุกบรรทัด

app.post("/books", (req, res) => {
  const book = {
    title: req.body.title,
    author: req.body.author,
    year: req.body.year
  };
});

✅ ดึงครั้งเดียว

app.post("/books", (req, res) => {
  const { title, author, year } = req.body;
  const book = { title, author, year };  // shorthand property
});

2. Guard clause + early return ใน validation

❌ if ครอบ logic ทั้งก้อน

if (title) {
  // ...โค้ดยาว 20 บรรทัด
  res.status(201).json(book);
} else {
  res.status(400).json({ error: "..." });
}

✅ ดักผิดก่อน เนื้อหลักไม่ย่อหน้า

if (!title) {
  return res.status(400).json({ error: "..." });
}
// ...เนื้อหลักอยู่ระดับบนสุด อ่านง่าย
res.status(201).json(book);

3. แยกไฟล์ตามหน้าที่ (server.js ต้องบาง)

❌ ทุกอย่างใน server.js

// server.js 300 บรรทัด
app.get("/books", (req,res)=>{ /*logic*/ });
app.post("/books", (req,res)=>{ /*logic*/ });
app.get("/members", ...);
// ...ปนกันหมด

✅ ผูก route → controller

// server.js (บาง)
app.use("/api/books", booksRouter);

// routes/books.routes.js
router.get("/", booksController.list);
router.post("/", booksController.create);
ใช้ใน Mission library-system เลย: 3 รูปแบบนี้คือเกณฑ์ "โครงสร้าง 30%" + "validation 20%" ของ review สัปดาห์นี้พอดี — เขียนแบบ ✅ ตั้งแต่ commit แรก
