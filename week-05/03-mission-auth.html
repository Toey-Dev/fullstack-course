🔐 ใส่ Auth ให้ library-system + test

เปลี่ยน library-system เป็นระบบที่มี "ผู้ใช้จริง" — portfolio piece สาย security ที่ทรงพลังที่สุด

ทำไม Mission นี้คือดาวเด่นใน portfolio

หลังจบ Mission นี้ library-system จะมี: สมัครสมาชิก, login, ป้องกัน route, แยกสิทธิ์ admin/user, ยืม-คืนแบบ transaction — ครบทุกอย่างที่ระบบจริงมี · และเพราะคุณเป็นอดีต pentester คุณจะทำ "ถูกหลัก security" ตั้งแต่แรก (hash, ไม่รั่ว, atomic) ซึ่งเป็นเรื่องเล่า interview ที่ dev ทั่วไปทำไม่ได้

เรียนคู่ Kcode Day 5 (JWT + Borrow/Return) ได้ — เขียนเองทุกบรรทัด ปิด AI

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 5.1 — Register + Login 75 นาที

  1. ตาราง users (id, email unique, password, role default 'user')
  2. POST /register — Zod validate → bcrypt.hash → INSERT (เช็ค email ซ้ำ → 409)
  3. POST /login — หา user → bcrypt.compare → sign JWT → คืน token (error ตอบกลาง ๆ)
security checklist: ไม่เก็บ plaintext · ไม่ส่ง password กลับ · login error ไม่บอกว่าผิดตรงไหน · JWT_SECRET ใน .env

Mission 5.2 — Auth middleware + protect 60 นาที

  1. middlewares/auth.js — ตรวจ Bearer token → แปะ req.user (401 ถ้าไม่ผ่าน)
  2. requireRole("admin") — เช็ค role (403 ถ้าไม่มีสิทธิ์)
  3. GET /me (login), DELETE /books/:id (admin) — ใส่ middleware
✅ เทสต์มือ: เรียก /me ไม่มี token → 401 · user ลบหนังสือ → 403 · admin ลบ → 200

Mission 5.3 — Borrow / Return (transaction) 75 นาที

  1. ตาราง borrows (id, user_id, book_id, borrowed_at, returned_at)
  2. POST /borrow — transaction: เช็คว่าง (FOR UPDATE) → INSERT borrow → UPDATE available=false
  3. POST /return — transaction: UPDATE returned_at → UPDATE available=true
คิดแบบ pentester: ลองยืมเล่มเดียวกัน 2 ครั้งติด → ครั้งที่สองต้องโดนปฏิเสธ (ไม่ใช่ยืมซ้ำได้)

Mission 5.4 — test auth flow + push 60 นาที

  1. test: register → login ได้ token → เรียก protected ผ่าน
  2. test เคสปฏิเสธ: ไม่มี token → 401 · user ลบ → 403 · ยืมซ้ำ → error
  3. npm test เขียวทั้งหมด · push — README + AI Usage
✅ เทสต์: test ครอบทั้งเคสผ่านและโดนปฏิเสธ — พิสูจน์ว่ายามกันได้จริง
