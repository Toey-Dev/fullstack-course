Week 18 — OAuth 2.0 / OIDC + Token Storage

"Login with Google" ทำงานยังไง — และเก็บ token ให้ปลอดภัยจริง (ปิดประเด็นค้างจาก Week 1)

🎯 เป้าหมายของสัปดาห์นี้

Week 5 คุณทำ login เอง (email+password+JWT) — Week 18 เรียน OAuth 2.0 / OIDC: ให้ผู้ใช้ login ด้วย Google/GitHub (ไม่ต้องจัดการ password เอง) · และปิดประเด็น "token เก็บที่ไหน" ที่คุยมาตั้งแต่ Week 1 ให้จบ — access token, refresh token, rotation, storage ที่ถูกต้อง · นี่คือความรู้ auth ระดับลึกที่ตำแหน่ง security ถามแน่นอน

  • OAuth 2.0 flow (Authorization Code + PKCE) — "Login with Google" ทำงานยังไง
  • OIDC = OAuth + identity (id_token) ต่างจาก OAuth ยังไง
  • access token vs refresh token + rotation
  • token storage ที่ถูกต้อง (สรุปจบจาก Week 1+5+8)
  • 🔐 ช่องโหว่ OAuth ที่พบบ่อย (redirect_uri, state/CSRF, token leak)
ทำไมสัปดาห์นี้สำคัญต่อสาย security: OAuth/OIDC เป็นมาตรฐาน auth ของทุกระบบใหญ่ — และมีช่องโหว่เฉพาะที่ pentester ต้องรู้ (open redirect, CSRF ผ่าน state, token in URL) · เข้าใจลึก = โดดเด่นใน interview AppSec
