Week 13 · Mission ใหญ่

⚙️ CI Pipeline ให้ library-system

ให้ GitHub Actions ตรวจโค้ดทุก PR — push แล้วเห็น check เขียว/แดงจริง

ทำไม Mission นี้ทำให้ repo ดูมืออาชีพทันที

หลังจบ library-system จะมี CI ที่รัน lint→test→build ทุก PR + badge เขียวใน README + branch protection · นี่คือสิ่งที่ recruiter เห็นแล้วรู้ทันทีว่า "ทำงานในทีมเป็น" · และเป็นรากของ CD (deploy อัตโนมัติ) ใน Week 14

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 13.1 — Workflow lint/test/build 75 นาที

  1. สร้าง .github/workflows/ci.yml — on push main + PR
  2. steps: checkout → setup-node (cache) → npm ci → lint → test → build
  3. เรียง fail fast (lint เร็วก่อน build ช้า) · pin action @v4
✅ เทสต์: push → ไป Actions tab → workflow รัน → เขียว

Mission 13.2 — PostgreSQL service สำหรับ test 60 นาที

  1. เพิ่ม services.postgres + health options
  2. ตั้ง DATABASE_URL ชี้ postgres service ใน test step
  3. test ที่ต้อง DB (integration จาก Week 3-5) ต้องเขียวใน CI
เทสต์จริง: integration test ที่คุย DB ต้องผ่านบน CI (ไม่ใช่แค่ในเครื่อง) — พิสูจน์ว่า setup ครบ

Mission 13.3 — Secrets + branch protection 45 นาที

  1. ย้าย secret (JWT_SECRET ฯลฯ) ไป GitHub Secrets → ใช้ ${{ secrets.X }}
  2. ตั้ง branch protection: require CI pass + PR review ก่อน merge main
  3. ทดสอบ: push ตรง main → ถูกบล็อก (ต้องผ่าน PR)
✅ เทสต์: สร้าง PR ที่ test พัง → merge ไม่ได้ (check แดง) · แก้ให้ผ่าน → merge ได้

Mission 13.4 — Badge + PR ทดสอบจริง 40 นาที

  1. เพิ่ม CI badge ใน README (![CI](.../badge.svg))
  2. สร้าง PR จริง 1 อัน (แก้เล็ก ๆ) → ดู CI รัน → merge เมื่อเขียว
  3. push + AI Usage
✅ เทสต์: README มี badge เขียว · workflow history มี run ที่ผ่าน PR จริง
