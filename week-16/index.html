Week 16 — Automation Report เป็น TS Service

เอางานจริงที่คุณทำมือทุกเดือนที่ media-presto มาทำเป็น service — portfolio ที่ "ใช้งานจริงในบริษัท"

🎯 เป้าหมายของสัปดาห์นี้ (portfolio พิเศษ)

สัปดาห์นี้ต่างจากที่ผ่านมา — แทนที่จะทำ library-system (โจทย์เรียน) คุณจะ rebuild งานจริงของคุณเอง: automation report ที่ดึง Meta/LINE/IG/GA4 มาสรุป ที่ตอนนี้ทำมือทุกเดือน · ทำเป็น TypeScript service ที่รันอัตโนมัติ โดยใช้ ทุกอย่างที่เรียนมา 15 สัปดาห์ · นี่คือ portfolio piece ที่ทรงพลังที่สุด เพราะเล่า interview ได้ว่า "แก้ปัญหาจริงในบริษัท + ลดเวลาทำงานจาก X ชม. เหลือ Y นาที"

  • ออกแบบ integration service: ดึงข้อมูลจากหลาย API ภายนอก (typed client)
  • Resilience: retry + backoff, timeout, Promise.allSettled (บางแหล่งล่มก็ยังออก report)
  • Rate limiting + caching (Redis) — ไม่ชน rate limit ของ API ภายนอก
  • Scheduled jobs (รันเองรายเดือน/วัน) + report export (JSON/CSV)
  • 🔐 จัดการ API key ปลอดภัย (ไม่ commit, ไม่ log) — มุม pentester
ทำไมสัปดาห์นี้คือ portfolio ที่ดีที่สุด: "ผลงานที่ใช้จริงในบริษัท" หนักกว่าโปรเจกต์เรียนหลายเท่า · และคุณวัดผลเป็นตัวเลขได้ (เวลาที่ประหยัด) ซึ่งเป็นภาษาที่ interviewer และ resume ชอบที่สุด

⚠️ เรื่อง confidential: ใช้ข้อมูล/โครงสร้างจริงเป็นแรงบันดาลใจได้ แต่ repo นี้ต้องไม่มี: API key จริงของลูกค้า, ข้อมูลลูกค้าจริง, business logic ลับของ media-presto · ทำเป็นเวอร์ชัน generic (mock data / sample) สำหรับ portfolio · ถ้าจะใช้ของจริงในงาน เก็บเป็น private repo แยก

🧹 Clean Code ประจำ Week 16

1. แต่ละ API = client แยก (interface เดียวกัน)

❌ fetch ทุก API ปนใน 1 ฟังก์ชัน

async function getReport() {
  const meta = await fetch(metaUrl)...
  const ga4 = await fetch(ga4Url)...
  // 200 บรรทัด ปนกันหมด แก้ยาก
}

✅ client แยก + interface เดียว

interface MetricsClient {
  fetch(clientId: string): Promise<Metrics>;
}
class MetaClient implements MetricsClient { ... }
class GA4Client implements MetricsClient { ... }
// เพิ่มแพลตฟอร์มใหม่ = client ใหม่ ไม่แตะของเดิม

2. แยก fetch / transform / export (3 ชั้น)

ดึงข้อมูล (client) → แปลง/สรุป (transformer) → ส่งออก (exporter) — แยกหน้าที่ test/แก้ง่าย (เหมือน layered architecture ฝั่ง API)

กฎทอง: client แยกต่อ API (interface เดียว, เพิ่มง่าย) · resilient (retry/timeout/allSettled) · cache external · ไม่ commit/log API key · วัดผลได้ (log เวลาที่ใช้ vs ทำมือ)
