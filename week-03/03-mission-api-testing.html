🧪 ใส่ Test ให้ library-system (Supertest)

เปลี่ยน library-system จาก "เทสต์มือใน Postman" เป็น "เทสต์อัตโนมัติ 1 คำสั่ง" — ก้าวที่ทำให้โค้ดไล่ทัน resume

ทำไม Mission นี้เปลี่ยนเกม

หลังจบ Mission นี้ พิมพ์ npm test ครั้งเดียว = เช็คทุก endpoint อัตโนมัติว่ายังทำงานถูก — นี่คือสิ่งที่ทำให้ตอน Week 4 (เพิ่ม DB), Week 6 (แปลง TypeScript) คุณ กล้าแก้ เพราะ test เป็นตาข่ายรองรับ และเป็นหลักฐานจริงของคำว่า "API testing" ใน resume

เรียนคู่ Kcode Day 3 (API testing drills) ได้ — แต่ เขียนเองทุกบรรทัด ปิด AI

📋 Supertest คืออะไร

Supertest = เครื่องมือยิง request เข้า Express app โดยไม่ต้องเปิด server จริง (เร็วและไม่ชนกัน) — เหมือน Postman อัตโนมัติที่เขียนเป็นโค้ด:

npm install -D supertest

// books.test.js
import request from "supertest";
import app from "../src/app.js";   // ⚠️ ต้อง export app แยกจาก listen

describe("GET /api/books", () => {
  it("คืน 200 + array ของหนังสือ", async () => {
    const res = await request(app).get("/api/books");
    expect(res.status).toBe(200);
    expect(res.body.ok).toBe(true);
    expect(Array.isArray(res.body.data)).toBe(true);
  });
});
เตรียมโครงก่อน: แยก app.js (สร้าง+config app, export) ออกจาก server.js (app.listen) — เพราะ test ต้อง import app มายิง request โดยไม่ start server จริง นี่คือ refactor แรกของ Mission

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 3.1 — Refactor + Error Middleware 45 นาที

  1. แยก src/app.js (export app) ออกจาก src/server.js (listen)
  2. ใส่ centralized error handler + 404 handler จากบทที่ 1 (ลำดับถูก)
  3. แปลง route ที่มี try/catch ซ้ำ → ใช้ next(err)
✅ เทสต์มือ: เรียก path มั่ว → ได้ JSON 404 / ทำให้ route พัง → ได้ JSON 500 ไม่ใช่ HTML

Mission 3.2 — Setup Vitest + test GET 60 นาที

  1. ติดตั้ง vitest + supertest, ตั้ง script "test": "vitest"
  2. เขียน tests/books.test.js — test GET /api/books (200 + array) และ GET /api/books/:id เคสเจอ (200)
  3. รัน npm test → เขียวผ่าน
✅ เทสต์: npm test ขึ้นเขียวอย่างน้อย 2 เคส

Mission 3.3 — test เคส error (404/400) 60 นาที

  1. GET /api/books/99999 (ไม่มี) → คาดหวัง 404 + ok: false
  2. POST /api/books body ว่าง → คาดหวัง 400
  3. POST /api/books ข้อมูลถูก → คาดหวัง 201 + book ที่สร้าง
คิดแบบ pentester: เพิ่ม test ส่ง field แปลก ๆ (title เป็นตัวเลข, ส่ง field เกิน, body ใหญ่มาก) — เคส "ป่วน" ที่ระบบจริงต้องทน
✅ เทสต์: test เคส error ทั้งหมดเขียว — พิสูจน์ว่า error handling ทำงานจริง

Mission 3.4 — Coverage + push 45 นาที

  1. เพิ่ม script "test:cov": "vitest run --coverage" (อาจต้อง npm i -D @vitest/coverage-v8)
  2. รันดู coverage — เป้า endpoint หลัก ≥ 70%
  3. เขียน unit test ฟังก์ชัน helper อย่างน้อย 1 ตัว (เช่น validate) ครบ 3 กลุ่ม (happy/edge/error)
  4. push library-system — README เพิ่ม badge/หัวข้อ "Testing" + section AI Usage
✅ เทสต์: npm test เขียวทั้งหมด · coverage ≥ 70% · repo มี test จริงบน GitHub
