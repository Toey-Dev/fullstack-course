🧹 Clean Code ประจำ Week 3 — เขียน test ที่ดี

1. ชื่อ test บอก "พฤติกรรม" ไม่ใช่ "ฟังก์ชัน"

❌ ชื่อคลุมเครือ

test("getById", () => { ... });
test("test1", () => { ... });

✅ อ่านแล้วรู้ว่าทดสอบอะไร

test("คืน 404 เมื่อหา book id ไม่เจอ", () => {});
test("คืน 201 + book เมื่อ POST ข้อมูลถูก", () => {});

2. โครง AAA — Arrange / Act / Assert

test("บวกเลขถูกต้อง", () => {
  const a = 2, b = 3;              // Arrange — เตรียมข้อมูล
  const result = add(a, b);        // Act — เรียกสิ่งที่ทดสอบ
  expect(result).toBe(5);          // Assert — ตรวจผล
});

3. test แต่ละอันต้องอิสระ (ไม่พึ่งกัน)

test ต้องรันสลับลำดับแล้วยังผ่าน · ไม่แชร์ state ระหว่าง test · ทดสอบ "1 พฤติกรรม 1 test" — ผิดแล้วรู้ทันทีว่าพังตรงไหน

กฎทอง: test คือ "เอกสารที่รันได้" — คนอ่าน test แล้วเข้าใจว่าโค้ดควรทำงานยังไงโดยไม่ต้องอ่าน implementation
