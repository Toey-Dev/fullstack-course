Week 19 — Security ใน CI (DevSecOps)

ให้ pipeline ตรวจ security ทุก PR อัตโนมัติ — เปลี่ยน security จาก "งานปลายทาง" เป็น "ตรวจตั้งแต่ต้น"

🎯 เป้าหมายของสัปดาห์นี้

Week 17-18 ป้องกันด้วยมือ — Week 19 ให้ CI ตรวจ security อัตโนมัติ: dependency มีช่องโหว่ไหม (npm audit), โค้ดมี pattern อันตรายไหม (SAST/Semgrep), เผลอ commit secret ไหม (secret scanning) · ฝัง security เข้า pipeline (Week 13) → PR ที่มีช่องโหว่ถูกจับก่อน merge · นี่คือหัวใจของ DevSecOps — ตำแหน่งสาย B ของคุณโดยตรง

  • Shift Left — ตรวจ security ตั้งแต่ต้น (ถูกกว่าเจอตอน production)
  • Dependency scanning (npm audit, Dependabot) — A06 OWASP
  • SAST (Semgrep) — หา pattern อันตรายในโค้ด (SQLi, XSS, secret)
  • Secret scanning (gitleaks) — กัน secret หลุดเข้า repo
  • รวมเข้า GitHub Actions — security gate ก่อน merge
ทำไมสัปดาห์นี้คือ DevSecOps แท้: "ฝัง security เข้า CI/CD" คือนิยามของ DevSecOps — ทักษะที่ JD ตำแหน่งนี้ขอตรง ๆ · คุณที่เข้าใจทั้ง CI (W13) + security (W17-18) = ทำ DevSecOps ได้จริง ซึ่งหายากมาก

- run: npm audit --audit-level=high
- uses: gitleaks/gitleaks-action
- uses: semgrep/semgrep-action
# ทุก PR ถูกสแกน ลืมไม่ได้

2. จัดการ false positive อย่างมีหลักการ

scanner เตือนผิดได้ (false positive) — ไม่ใช่ปิด scan ทั้งหมด แต่ ignore เฉพาะจุด + comment เหตุผล (audit trail) · มุม pentester: ประเมินว่าจริงไหม

กฎทอง: security scan ทุก PR (gate) · fail บน High/Critical · จัดการ false positive รายจุด (ไม่ปิดทั้งหมด) · secret scan รวม git history · update dependency สม่ำเสมอ (Dependabot)
