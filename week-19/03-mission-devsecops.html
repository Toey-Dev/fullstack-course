Week 19 · Mission ใหญ่

🔒 Security Pipeline ให้ library-system

ฝัง security scanning เข้า CI — ทุก PR ถูกตรวจอัตโนมัติ (DevSecOps จริง)

ทำไม Mission นี้คือ portfolio DevSecOps

หลังจบ library-system จะมี security pipeline (dependency + secret + SAST) เป็น gate ใน CI · นี่คือหลักฐานว่าคุณ "ฝัง security เข้า CI/CD เป็น" — ทักษะที่ JD DevSecOps ขอตรง ๆ · บวกกับ pentest background ของคุณ = โปรไฟล์ AppSec/DevSecOps ที่สมบูรณ์

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 19.1 — Dependency + Secret Scan 60 นาที

  1. เพิ่ม npm audit --audit-level=high ใน CI (W13) + Dependabot config
  2. gitleaks-action ใน CI + pre-commit hook (husky + gitleaks protect)
  3. รัน → ดูผล → แก้/rotate ถ้าเจอ
✅ เทสต์: ลอง commit secret ปลอม → pre-commit/CI จับได้ · npm audit เขียว (หรือแก้จนเขียว)

Mission 19.2 — Semgrep SAST 60 นาที

  1. semgrep-action ใน CI (config: p/owasp-top-ten p/javascript)
  2. รัน → อ่านผล → แก้ที่เจอจริง
  3. (challenge) เขียน custom rule 1 ข้อ (เช่น บังคับ parameterized query)
มุม pentester: ลองใส่โค้ดมีช่องโหว่ (SQL ต่อ string) → Semgrep จับได้ → แก้

Mission 19.3 — Security Gate + False Positive 60 นาที

  1. รวม 3 scan เป็น security workflow + เป็น required check (branch protection)
  2. ทดสอบ: PR ที่มีช่องโหว่ → ถูกบล็อก merge
  3. จัดการ false positive ที่เจอ (ignore เฉพาะจุด + comment เหตุผล)
✅ เทสต์: PR ช่องโหว่ → check แดง merge ไม่ได้ · false positive ถูก ignore อย่างมีเหตุผล

Mission 19.4 — Report + push 45 นาที

  1. อัปเดต SECURITY.md: security tooling (audit/gitleaks/Semgrep) + ผลที่เจอ/แก้
  2. เพิ่ม security badge (ถ้ามี) · README อธิบาย DevSecOps setup
  3. push + AI Usage
✅ เทสต์: SECURITY.md อ่านแล้วเห็น "ระบบมี security scanning อะไรบ้าง ทำงานยังไง"
