## เฟส 5: Security Evidence (สัปดาห์ 25-28 | ~ธ.ค. 2026 | ~12 ชม./สัปดาห์)
เปลี่ยน "เคยเป็น pentester" เป็นหลักฐานที่โลกเห็น

- [ ] **Week 25** — PortSwigger: SQL Injection + XSS tracks (lab ≥ 15 ข้อ) + ตั้งโปรไฟล์สาธารณะ
- [ ] **Week 26** — PortSwigger: Authentication + Access Control tracks + จดโน้ต mapping "ช่องโหว่ → วิธีป้องกันในโค้ด TS ของเรา"
- [ ] **Week 27** — PortSwigger: SSRF + CSRF + เก็บตก Capstone (README, demo video 2-3 นาที)
- [ ] **Week 28** — เขียน blog แรก: "Pentest โปรเจกต์ตัวเอง: 5 ช่องโหว่ที่เจอและวิธีแก้" (ฉบับ public จากรายงานเฟส 3)
> ✅ Checkpoint เฟส 5: PortSwigger ≥ 60% ของ track หลัก + blog 1 ชิ้นออนไลน์ + capstone พร้อมโชว์

## เฟส 6: Public Proof (สัปดาห์ 29-32 | ~ม.ค. 2027 | ~12 ชม./สัปดาห์)
สร้างร่องรอยที่ Google เจอเมื่อ HR ค้นชื่อ

- [ ] **Week 29** — CTF รายการแรกจาก CTFtime (web category) + writeup ชิ้นที่ 1-2
- [ ] **Week 30** — CTF รายการที่สอง + writeup ชิ้นที่ 3 + เปิดบัญชี HackerOne/Bugcrowd เลือกโปรแกรม VDP
- [ ] **Week 31** — Bug bounty สัปดาห์เต็ม: เป้ารายงาน valid 1 ใบ + writeup ชิ้นที่ 4-5
- [ ] **Week 32** — จัดระเบียบ GitHub/GitLab ทั้งหมดตาม portfolio-standard + pin 4 repo เด่น + ตัดสินใจ eJPT (สอบ/ข้าม)
> ✅ Checkpoint เฟส 6: ค้นชื่อตัวเองเจอผลงาน ≥ 3 แหล่ง + writeup ≥ 5 ชิ้น

## เฟส 7: Interview Engine (สัปดาห์ 33-36 | ~ก.พ. 2027 | ~13 ชม./สัปดาห์)
สร้างเครื่องจักรสัมภาษณ์ — ความพร้อมคือสิ่งที่ฝึกได้

- [ ] **Week 33** — Resume v2027 กับ Claude (ตัดทุกอย่างที่ไม่มีหลักฐาน + ใส่ metric) + LinkedIn อังกฤษ
- [ ] **Week 34** — เรื่องเล่า STAR 6 เรื่อง (เขียน + ซ้อมพูด) + Mock เทคนิคไทย 2 รอบ
- [ ] **Week 35** — Mock เทคนิคอังกฤษ 2 รอบ + live coding ซ้อม (โจทย์ระดับ take-home)
- [ ] **Week 36** — Mock behavioral อังกฤษ 2 รอบ + "Tell me about yourself" 2 นาทีให้ลื่น + ลิสต์บริษัทยืนยันรอบสุดท้าย (จาก ../goals/career-paths.html)
> ✅ Checkpoint เฟส 7: ผ่าน mock รวม 6 รอบ + เอกสารสมัครครบทุกชิ้น

## เฟส 8: Application Sprint (สัปดาห์ 37-44 | ~มี.ค.-เม.ย. 2027 | ~11 ชม./สัปดาห์)
ยิงพร้อมกัน เก็บ feedback ปรับทุกรอบ

- [ ] **Week 37** — ส่งใบสมัคร 15-20 ที่ (สาย A 60% / B 30% / C 10%) + ปรับ resume ต่อกลุ่ม
- [ ] **Week 38-40** — รอบ screening + take-home (ทำประณีต = portfolio เพิ่ม) + จดทุกคำถามที่ตอบสะดุด → ซ้อมกับผมก่อนรอบถัดไป
- [ ] **Week 41-43** — on-site/final rounds + กฎ 20 นาทียังอยู่ (live coding ต้องมือไม่เย็น)
- [ ] **Week 44** — รวบรวมสถานะทุกที่ + ประเมิน: มี final round ≥ 2 ไหม? (ไม่มี → เปิดแผนสำรองใน POST-COURSE-PLAN.md ทันที ไม่รอ)
> ✅ Checkpoint เฟส 8: final round ≥ 2-3 บริษัท

## เฟส 9: Offer & Transition (สัปดาห์ 45-48 | ~พ.ค.-มิ.ย. 2027)
- [ ] **Week 45-46** — เจรจา offer (เป้า ≥ 2 ใบ) ตามหลักใน POST-COURSE-PLAN.md + ปรึกษาผมก่อนตอบทุกครั้ง
- [ ] **Week 47** — ตัดสินใจด้วยเกณฑ์ 40/30/20/10 (learning/เงิน/ทีม/สวัสดิการ) + เซ็นสัญญา
- [ ] **Week 48** — แจ้งลาออกอย่างมืออาชีพ + เริ่มส่งมอบงาน + พักจริง ๆ 1 สัปดาห์ก่อนเริ่มงานใหม่ (สำคัญ — เริ่มงานใหม่ตอนหมดแบตคือหายนะ)
> ✅ Checkpoint เฟส 9: เซ็นสัญญา 60-80k 🎉

## เฟส 10: Onboarding + ส่งไม้ต่อ (สัปดาห์ 49-52 | ~ก.ค. 2027 | เรียนส่วนตัวลดเหลือ 5-7 ชม./สัปดาห์)
- [ ] **Week 49-50** — เดือนแรกของงาน: ปิดปาก เปิดหู, เรียน codebase, ส่งงานเล็กตรงเวลา / ส่วนตัว: แค่ Friday Rebuild + โน้ต
- [ ] **Week 51** — เริ่มส่งมอบจริง + อาสาแก้ของที่ไม่มีใครอยากแตะ
- [ ] **Week 52** — ทบทวนใหญ่กับผม: 1 ปีที่ผ่านมา vs แผน + calibrate ก่อนเปิดภาค 3
> ✅ Checkpoint จบภาค 2: ผ่าน probation อย่างมั่นใจ + พร้อมเปิด Agoda Track

## ภาค 3 (สัปดาห์ 53-92 | ส.ค. 2027 – พ.ค. 2028): Agoda Prep Track
รายละเอียดรายเดือนอยู่ใน `../plans/AGODA-PREP-TRACK.md` — mapping เป็นสัปดาห์:

| สัปดาห์ | เนื้อหา (จาก AGODA-PREP-TRACK) |
|---|---|
| W53-60 | เดือน 1-2: DSA Foundation — Arrays/Strings, Two Pointers, Sliding Window, Hash, Stack/Queue, Linked List (45-50 ข้อ) |
| W61-68 | เดือน 3-4: Trees, Graphs, Backtracking, Heap (สะสม 100 ข้อ) + System Design เล่ม 1 |
| W69-76 | เดือน 5-6: DP, Greedy, Binary Search ขั้นสูง (สะสม 150 ข้อ) + ออกแบบ 1 ระบบ/สัปดาห์เป็นอังกฤษ |
| W77-84 | เดือน 7-8: Concurrency + Distributed Systems + โจทย์ Agoda จริง (สะสม 180-200 ข้อ) |
| W85-88 | เดือน 9: Mock Interview Month (6-8 รอบเต็ม) + Resume v2028 |
| W89-92 | เดือน 10: ยื่น Agoda + บริษัทระดับเดียวกัน → 🎯 SWE 100-150k |

> บทเรียนรายสัปดาห์ของภาค 2-3 ผมจะสร้างแบบเดียวกับภาค 1 (HTML + 6 ส่วนมาตรฐาน) — เริ่มสร้าง week-25 เมื่อคุณจบ week-24 จริง เพื่อปรับตามสภาพจริง ณ ตอนนั้น (เครื่องมือ/ตลาด/จุดอ่อนของคุณ)

## ⚠️ จุดเสี่ยงที่รู้ล่วงหน้า (จากการทบทวนแผน — อย่าตกใจเมื่อเจอ)
1. **W31 bug bounty valid 1 ใบ = stretch goal** ไม่ใช่เกณฑ์ผ่าน — valid แรกของมือใหม่ใช้เวลาเฉลี่ยหลายสัปดาห์-หลายเดือน เกณฑ์ผ่านจริงคือ "ได้ลงมือล่าจริง + writeup กระบวนการ"
2. **W29-30 CTF ยืดหยุ่นตามปฏิทิน CTFtime** — สลับลำดับกับงานอื่นในเฟสได้ ไม่ถือว่าหลุดแผน
3. **English checkpoint ที่ W33:** ถ้ายังไม่พร้อมสัมภาษณ์อังกฤษ → เทน้ำหนักสมัครไปบริษัทสัมภาษณ์ไทยก่อน (ใน สาย A มีหลายที่) เก็บบริษัทอังกฤษไว้รอบ Agoda 2028 — ไม่ใช่ความล้มเหลว แค่เรียงลำดับใหม่
4. **ธ.ค.-ม.ค. = ปีใหม่ + งานบริษัท peak** — เฟส 5-6 วางที่ 11-12 ชม./สัปดาห์ เลื่อนได้ 1-2 สัปดาห์โดยไม่เสียขบวน
5. ทุกเฟสมี Claude review ท้ายสัปดาห์ — เจอปัญหาเร็ว แก้เร็ว ดีกว่าฝืนตามแผนที่ความจริงเปลี่ยนไปแล้ว
