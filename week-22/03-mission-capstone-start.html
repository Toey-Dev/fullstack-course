Week 22 · Mission ใหญ่

🏗 Capstone: Design Doc + Scaffold + Vertical Slice แรก

วางรากฐานโปรเจกต์ปิดคอร์ส — ออกแบบ + ตั้งโครง + 1 feature ทำงานจริง

เป้าหมาย Mission สัปดาห์นี้

จบสัปดาห์นี้คุณจะมี: repo capstone ที่มี design doc + threat model, scaffold ที่ CI เขียว + Docker รัน, และ 1 feature ทำงานครบ stack + deploy ได้ · นี่คือรากฐานที่ Week 23 จะต่อยอดให้เสร็จ · เลือก VulnTracker (แนะนำ — ตรงงานคุณ เล่า interview ดี) หรือ SecureShare

หลักประจำสัปดาห์: design พอเห็นภาพแล้วเริ่ม (ไม่ over-design) · MVP ก่อน · vertical slice (ครบชั้น+deploy) · apply ทุกอย่างที่เรียน · threat model ตั้งแต่ออกแบบ

🪜 4 Mission ย่อย

Mission 22.1 — Design Doc + Threat Model 90 นาที

  1. เลือก capstone + สร้าง repo ใหม่ (public)
  2. เขียน docs/DESIGN.md: Problem, Requirements (func/non-func), Architecture diagram, Data model, API list
  3. เขียน docs/THREAT-MODEL.md: ตาราง STRIDE 6 แถว + วิธีป้องกันแต่ละข้อ
✅ เช็ก: ใครอ่าน DESIGN.md แล้วเข้าใจระบบใน 5 นาที · threat model ครบ 6 STRIDE

Mission 22.2 — Scaffold + CI เขียว 90 นาที

  1. โครง repo (apps/api, apps/web, docs) + TS strict + ESLint/Prettier
  2. docker-compose.yml (api, web, postgres, redis) รันขึ้นได้
  3. GitHub Actions CI: lint + test + build + (gitleaks/npm audit) → เขียว
  4. 1 test ผ่าน (health check) + commit แรกแบบ conventional
เช็ก: docker compose up รันได้ · CI badge เขียวบน README · ไม่มี secret ใน repo (gitleaks ผ่าน)

Mission 22.3 — Auth + Vertical Slice แรก 2 ชม.

  1. auth (register/login JWT + bcrypt) + RBAC middleware
  2. feature หลัก 1 ตัวครบชั้น: migration+index → Zod → route+RBAC → service → test (UI พื้นฐานก็ได้)
  3. audit log การ action ที่เปลี่ยนข้อมูล (defense ตาม threat model)
✅ เช็ก: สร้าง record ได้ผ่าน API · test RBAC ผ่าน (role ผิด → 403) · audit log บันทึก

Mission 22.4 — Deploy + README + push 75 นาที

  1. deploy ขึ้น (Vercel web + Railway/Render api+db) — feature แรกใช้งานได้บน production
  2. README: วิธีรัน local + architecture diagram + screenshot + live URL
  3. push ทุกอย่าง · AI Drill #21 (ให้ AI review design doc + ฝึก system design 1 ข้อ EN)
✅ เช็ก: live URL เปิดได้ · README ครบ (recruiter อ่านแล้วเข้าใจ) · ทุกอย่างบน GitHub
