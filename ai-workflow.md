# 🤖 AI Workflow — ใช้ AI เป็น "ตัวคูณ" ไม่ใช่ทางลัด

> เป้าหมาย: เป็น dev ที่ "ใช้ AI ได้เร็ว และตรวจงาน AI เป็น" — ทักษะที่บริษัท 2026 จ่ายแพง

## 1. กฎสองโหมด (ใช้ตัดสินใจทุกครั้งก่อนเปิด AI)

| | 🎓 โหมดเรียน | ⚡ โหมดทำงาน |
|---|---|---|
| ใช้เมื่อ | แนวคิดที่**เพิ่งเรียน / ยังไม่เคยเขียนเองจนคล่อง** | ของที่เข้าใจแล้ว เคยเขียนเองมาแล้ว |
| AI ทำอะไรได้ | เป็นติวเตอร์: ถามคอนเซ็ปต์, ขอ hint ทีละชั้น, ตรวจโค้ดหลังเขียนเสร็จ, อธิบาย error | เต็มสปีด: boilerplate, scaffold test, refactor, แปลง syntax, เขียน docs |
| AI ห้ามทำ | **เขียนโค้ดแทน** | — (แต่ต้อง review ทุกบรรทัดก่อน commit) |
| ตัวอย่าง | Mission ในคอร์สนี้ทุกข้อ = โหมดเรียน | งาน WordPress ที่ media-presto, HTML/CSS ที่คล่องแล้ว |

**เกณฑ์ตัดสินใจ 1 คำถาม:** "ถ้า AI เขียนผิด ฉันจะมองออกไหม?" — มองออก = โหมดทำงาน / ไม่แน่ใจ = โหมดเรียน

## 2. AI Code Review Drill (ทุกสัปดาห์ 30 นาที — คู่กับ Friday Rebuild)

ทักษะ "จับผิดโค้ด AI" คือจุดที่ pentester อย่างวีได้เปรียบที่สุดในตลาด

**วิธีทำ:**
1. ขอให้ AI (Claude/อื่น ๆ) เขียนโค้ดตามโจทย์ประจำสัปดาห์ (ผมจะกำหนดให้ท้ายบทเรียนแต่ละสัปดาห์)
2. **ก่อนรัน** — อ่านหาบั๊ก, ช่องโหว่ security, edge case ที่พลาด ให้ได้อย่างน้อย 2 จุด
3. จดลง `notes/ai-review-log.md`: โจทย์ / จุดที่เจอ / AI พลาดแนวไหนบ่อย

**ตัวอย่างโจทย์ Drill แรก (Week 2):** "ขอ Express endpoint รับ POST /users บันทึกลง database"
— จุดที่ AI มักพลาด: ไม่ validate input, SQL injection (ถ้าต่อ string), ไม่ hash password, error message รั่วข้อมูลภายใน, ไม่มี rate limit

## 3. บันทึกการใช้ AI ใน Portfolio (template)

ใส่ section นี้ใน README ของทุก repo:

```markdown
## AI Usage
- Scaffolding & boilerplate: AI-assisted (Claude)
- Business logic & security review: เขียนและตรวจเองทั้งหมด
- ทุกบรรทัดผ่านการ review ก่อน commit
- จุดที่ AI เสนอแล้วผมแก้: [ตัวอย่างสั้น ๆ 1-2 จุด เช่น "AI ไม่ได้ validate input ผมเพิ่ม Zod"]
```

ข้อสุดท้ายสำคัญที่สุด — มันพิสูจน์ว่าคุณ "ตรวจเป็น" ไม่ใช่แค่ "ใช้เป็น" และเป็นเรื่องเล่าที่ดีมากใน interview

## 4. เครื่องมือที่ควรคล่อง (เรียงตามตลาดไทยต้องการ)

| เครื่องมือ | ใช้ทำอะไร | เมื่อไหร่ในแผน |
|---|---|---|
| Claude (chat/Cowork) | ติวเตอร์, review, ออกแบบ | ใช้อยู่แล้ว ✅ |
| GitHub Copilot | autocomplete ใน VS Code | เริ่ม Week 6 (หลัง TS พื้นฐานแน่น — ก่อนหน้านั้นปิดไว้ตอนทำ Mission) |
| Claude Code / CLI agents | งาน refactor ใหญ่, สร้าง test ทั้งชุด | เฟส 2 เป็นต้นไป |
| Gemini CLI (Kcode Day 10) | เรียนแนวคิด แล้วเทียบกับตัวอื่น | ตามคอร์ส |

**กฎเดียวที่ห้ามแหก:** ระหว่างทำ Mission ในคอร์ส = ปิด autocomplete/AI ทั้งหมด เขียนมือล้วน
เพราะ Mission คือเครื่องสร้างความจำระยะยาว — ปนเปื้อนเมื่อไหร่ ปัญหา "ลืมทักษะ" กลับมาทันที
