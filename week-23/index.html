🧹 หลักการประจำ Week 23

1. "Done" มีนิยาม — Definition of Done

❌ "done = ฉันเขียนเสร็จแล้ว"

// feature ขึ้นแล้ว แต่:
// ไม่มี test · error case พัง
// ไม่มี loading/empty state
// README ว่าง · ยังไม่ deploy

✅ done = ผ่าน checklist

// ✓ happy + edge + error case
// ✓ test ผ่าน · validate input
// ✓ loading/empty/error UI
// ✓ deploy ได้ · doc อัปเดต

2. Polish คือสิ่งที่คนจำได้ — แต่ทำหลัง core เสร็จ

empty state, loading spinner, error message ที่เป็นมิตร, responsive — สิ่งเล็ก ๆ ที่ทำให้ดู "เสร็จจริง" · แต่ทำหลัง core feature + security เสถียรแล้ว ไม่ใช่ polish ก่อนระบบทำงาน

กฎทอง: ทุก feature มี Definition of Done · test ครอบคลุม happy+edge+error · security/perf pass ก่อนเรียกว่าเสร็จ · polish UI หลัง core นิ่ง · case study เล่าเป็นเรื่อง (ปัญหา→ออกแบบ→trade-off→ผล)
