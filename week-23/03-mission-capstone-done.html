เป้าหมาย Mission สัปดาห์นี้

จบสัปดาห์นี้ capstone ของคุณ = portfolio piece สมบูรณ์: feature ครบ user journey, test ครอบคลุม, ผ่าน self-pentest, เร็วพอ (มีตัวเลข), polish แล้ว, มี case study เล่าได้ 5 นาที, tag v1.0 · พร้อมใส่ resume + โชว์ interview

หลักประจำสัปดาห์: Definition of Done ทุก feature · เทสต์ 3 มิติ (happy/edge/error) · self-pentest OWASP · measure ก่อน optimize · polish หลัง core นิ่ง · case study เป็นเรื่องเล่า

🪜 4 Mission ย่อย

Mission 23.1 — Should-have Features ครบ user journey 2 ชม.

  1. เพิ่ม features ที่ขาดให้ user journey ครบ (เช่น VulnTracker: audit log view, filter, export, assign)
  2. ทุก feature ผ่าน Definition of Done (happy/edge/error + validate + RBAC + UI state)
  3. UI มี loading / empty / error state ครบ
✅ เช็ก: user ทำ flow หลักได้ครบตั้งแต่ต้นจนจบ · ทุก feature มี error handling

Mission 23.2 — Test Coverage มีความหมาย 90 นาที

  1. เพิ่ม test ครอบคลุม logic สำคัญ (RBAC, validation, status transition, business rule)
  2. ทุก endpoint สำคัญมี integration test (happy + 403 + 400/404)
  3. 1-2 E2E flow หลัก (login → action → เห็นผล) · CI ยังเขียว
เช็ก: test จับจุดเสี่ยงจริง (ไม่ใช่ไล่ %) · CI เขียว · พังโค้ดดู test แดงจริง

Mission 23.3 — Security + Performance Pass 2 ชม.

  1. self-pentest ตาม OWASP checklist → เขียน docs/SECURITY-AUDIT.md (เจออะไร แก้ยังไง)
  2. แก้ช่องโหว่ที่เจอ (IDOR/injection/misconfig) + npm audit/gitleaks/Semgrep ผ่าน
  3. perf pass: index/N+1/cache/pagination → docs/PERFORMANCE.md (before/after)
✅ เช็ก: ไม่มีช่องโหว่ critical · มีตัวเลข perf พิสูจน์ · scan อัตโนมัติเขียว

Mission 23.4 — Polish + Case Study + Demo + v1.0 90 นาที

  1. polish UI (responsive, empty/loading/error, ขัดเงา) + screenshot
  2. เขียน docs/CASE-STUDY.md (Problem→Approach→Security→Perf→Result) + อัปเดต README
  3. ซ้อมเล่า demo 5 นาทีออกเสียง · push · git tag v1.0 · AI Drill #22
✅ เช็ก: live URL ใช้ได้ · README+case study ครบ · เล่า demo ได้ลื่น · tag v1.0 บน GitHub
