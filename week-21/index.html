// EXPLAIN ANALYZE หา query ช้า
// log เวลา → หา bottleneck จริง
// optimize → วัดซ้ำ (พิสูจน์เร็วขึ้น)

2. อย่า premature optimization

"premature optimization is the root of all evil" — optimize เมื่อมีปัญหาจริง (วัดแล้วช้า) ไม่ใช่ทุกที่ตั้งแต่แรก · code ที่อ่านง่าย > เร็วแต่ซับซ้อนโดยไม่จำเป็น

กฎทอง: measure ก่อน optimize (EXPLAIN/profiling) · index เฉพาะ query ที่ใช้จริง · cache มี invalidation เสมอ · งานหนัก→queue (ไม่ block request) · อธิบาย trade-off ได้ (ไม่มี free lunch)
