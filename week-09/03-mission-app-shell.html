Week 9 · Mission ใหญ่

🏗 สร้าง App Shell + Auth ให้ library-web

ยกระดับ library-web จาก "หน้าเดียว" สู่ "แอปเต็มระบบ" ที่มีโครงและ auth จริง

ทำไม Mission นี้ทำให้แอปดู "เสร็จจริง"

หลังจบ Mission นี้ library-web จะมี: หน้า login/register, ป้องกันหน้า, sidebar/topbar, แสดงข้อมูลผู้ใช้ — โครงเดียวกับแอป production ทุกตัว · ความรู้ pentester ทำให้คุณกัน access control ครบ 3 ชั้น (ที่ dev ทั่วไปมักลืมชั้น backend)

ต่อยอด library-web จาก Week 8 + ต้องมี backend (library-system) รันอยู่

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 9.1 — Auth pages 75 นาที

  1. route group (auth) + layout เต็มจอ
  2. หน้า /login + /register — form (Tailwind), validate, error กลาง ๆ, loading
  3. เชื่อม route handler proxy (token → HttpOnly cookie จาก Week 8)
✅ เทสต์: login สำเร็จ → เด้ง /dashboard · รหัสผิด → error กลาง ๆ · ปุ่ม disabled ตอน loading

Mission 9.2 — Protect routes (3 ชั้น) 60 นาที

  1. middleware.ts — กัน /dashboard, /books → ไม่มี token เด้ง /login (matcher ครบทุก path)
  2. login แล้วเข้า /login → เด้ง /dashboard
  3. ดัก 401 จาก API → เด้ง /login (token หมด)
security: ตรวจว่า backend ยัง requireRole อยู่ (ชั้น 3) — frontend กันแค่ UX

Mission 9.3 — App Shell 75 นาที

  1. route group (app) + layout (sidebar + topbar + content)
  2. Sidebar เมนู (Dashboard, Books, + เตรียม Users/Borrows) + active link
  3. Topbar — ดึง user จริงจาก /api/me + avatar (อักษรแรก) + ปุ่ม logout
✅ เทสต์: หน้าในระบบมี shell เหมือนกัน · เมนูปัจจุบันไฮไลต์ · topbar แสดงชื่อจริง · responsive (sidebar ยุบบนมือถือ)

Mission 9.4 — Logout + polish + push 45 นาที

  1. logout (ลบ cookie ที่ server) → เด้ง /login
  2. responsive: sidebar เป็น hamburger/drawer บนมือถือ
  3. push — README screenshot โครงแอป + AI Usage
✅ เทสต์: logout แล้วเข้า /dashboard ตรง → เด้ง /login · ใช้งานบนมือถือได้
