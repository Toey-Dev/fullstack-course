🎤 Mock Interview + เจรจาเงินเดือน

ซ้อมจนพูดได้ลื่น — technical, system design, เล่า capstone, ต่อรองเงินเดือน

1 · โครงสัมภาษณ์ dev — รู้ว่าจะเจออะไร

สัมภาษณ์ตำแหน่ง full stack ในไทย (บริษัท product / agency / บริษัทใหญ่อย่าง Agoda) มักมีรอบเหล่านี้:

รอบวัดอะไรคุณเตรียมจาก
HR / cultureทำไมอยากมา, แรงจูงใจ, soft skillเรื่องเล่าเส้นทาง: pentester→full stack→secure
Technical Q&AJS/TS, Node, DB, security พื้นฐานทั้งคอร์ส (W1-21) — ทบทวน quiz
Live codingแก้โจทย์สด + คิดออกเสียงDSA (Agoda Track) + เขียนสะอาด
System designออกแบบระบบ + trade-offW21-22 + AI drill system design
Project deep-diveเล่า + ขุดลึก capstonecase study + ป้องกันทุก resume bullet
หลัก: ไม่มีรอบไหนที่คุณไม่ได้เตรียมมา — คอร์สนี้ build มาเพื่อตอบทุกรอบ · สัปดาห์นี้แค่ซ้อมพูดสิ่งที่ทำมาแล้ว ให้ออกมาเป็นคำพูดที่ลื่น มั่นใจ

2 · เล่า Capstone ด้วย STAR — โครงที่ขุดยังไงก็ไม่หลุด

"Tell me about a project" คือคำถามที่ทุกสัมภาษณ์ถาม · ใช้โครง STAR + ลึกพอให้ขุดต่อได้:

STARVulnTracker ของคุณ
Situationทีม pentest บันทึก finding กระจัดกระจาย ติดตามสถานะยาก
Taskสร้างระบบ track finding ที่ปลอดภัย + มี RBAC + audit
Actiondesign doc + threat model, build TS/Express/Next, self-pentest, optimize
Resultlive system, แก้ IDOR ที่เจอ, dashboard 850→45ms, test coverage จุดเสี่ยง

เตรียมรับการขุดลึก (follow-up ที่จะเจอ)

  • "ทำไมเลือก PostgreSQL ไม่ใช่ MongoDB?" → relational data + ACID + ต้อง JOIN finding/project/user
  • "IDOR ที่เจอเป็นยังไง แก้ยังไง?" → เล่า ownership check ฝั่ง server + 404 anti-enumeration
  • "ถ้า scale 100x ต้องทำอะไร?" → read replica, cache layer, queue, partition (W21-22)
  • "จุดที่ภูมิใจ / จุดที่จะทำต่างออกไป?" → ความซื่อสัตย์ = น่าเชื่อถือ
💡 จุดแข็งคุณ: ส่วนใหญ่ candidate เล่า project ได้แค่ "ทำอะไร" · คุณเล่าได้ถึง threat model, ช่องโหว่จริงที่แก้, และตัวเลข perf — ลึกระดับที่ทำให้ interviewer จำได้

3 · System Design สด — เฟรมเวิร์กตอบ (Agoda Track)

รอบ system design กลัวกันมาก แต่มีสูตร · ใช้กรอบ 5 ขั้น (ตอบเป็นอังกฤษได้ตามที่ซ้อม AI drill):

① ถามให้ชัดก่อน (อย่ารีบวาด) · ② API + data model · ③ วาดภาพรวม · ④ จุด scale (apply W21!) · ⑤ trade-off + security (จุดขายคุณ)

เคล็ด: คิดออกเสียงตลอด — interviewer วัดวิธีคิดไม่ใช่คำตอบสมบูรณ์ · ถามกลับได้ · พูดถึง security เสมอ (คนอื่นลืม) · "measure ก่อน optimize" + "ไม่ over-engineer" = สัญญาณ senior

4 · เจรจาเงินเดือน — ตลาดไทย สาย Secure Full Stack

เป้าหมาย 100k ของคุณเป็นจริงได้ด้วยข้อมูล + การวางตัว · หลักเจรจา:

  • รู้ตลาดก่อน — ดูช่วงเงินเดือนจริง (เช็ค ISM Technology, Talent Sauce, JobsDB, LinkedIn Salary, เพื่อนในวงการ) ตามตำแหน่ง+ประสบการณ์
  • อย่าบอกตัวเลขแรก — ถ้าถามให้คืนคำถาม "ช่วงงบของตำแหน่งนี้เท่าไหร่ครับ" · ถ้าต้องบอก ให้ช่วงที่ปลายล่างคือเป้า
  • ขายคุณค่า ไม่ใช่ความต้องการ — "ด้วยพื้นเพ security + full stack ผมช่วยลดความเสี่ยงที่ปกติต้องจ้างแยก" ไม่ใช่ "ผมอยากได้ 100k เพราะ..."
  • มองทั้ง package — base + โบนัส + WFH + อุปกรณ์ + งบเรียน/cert + การเติบโต (เส้นทาง AI-Security)
  • มีหลายตัวเลือก = อำนาจต่อรอง — สมัครหลายที่พร้อมกัน · offer หนึ่งช่วยต่อรองอีกที่
⚠️ ไม่ใช่คำแนะนำการเงินส่วนบุคคล: ตัวเลขตลาดเปลี่ยนตลอด — เช็คแหล่งล่าสุดเองก่อนเจรจา · การกระโดดจาก 40k → 100k อาจต้อง 1-2 ก้าว (เปลี่ยนงานไป 60-70k ก่อน แล้วขยับ) ตามที่วางใน FIVE-YEAR-PLAN · ใช้ข้อมูลจริงตัดสินใจ

5 · ภาพรวม + คำช่วยจำ

เล่า project: STAR + ขุดลึกได้ · System design: Clarify→API→Diagram→Scale→Trade-off

เจรจา: รู้ตลาด · อย่าบอกก่อน · ขายคุณค่า · ดูทั้ง package

"interviewer วัดวิธีคิด ไม่ใช่คำตอบเป๊ะ · คิดออกเสียง · พูด security เสมอ · เจรจาด้วยข้อมูลไม่ใช่อารมณ์"

6 · Quiz

Q1. ในรอบ system design ทำไม "ถามให้ชัดก่อน" สำคัญกว่ารีบวาดทันที?
เฉลยเพราะ scope ที่ไม่ชัดทำให้ออกแบบผิดทาง · การถาม (จำนวน user, read/write ratio, ข้อจำกัด) แสดงว่าคุณคิดเป็นระบบและไม่เดา — เป็นสัญญาณ senior · interviewer มักจงใจให้โจทย์กว้างเพื่อดูว่าคุณถามไหม
Q2. ทำไมไม่ควรบอกตัวเลขเงินเดือนที่ต้องการก่อน?
เฉลยคนบอกตัวเลขก่อนมักเสียเปรียบ — ถ้าต่ำไปก็ปิดเพดานตัวเอง, สูงไปก็เสี่ยงถูกคัด · ให้ฝั่งบริษัทเผยช่วงงบก่อน หรือถ้าต้องบอกให้อิงข้อมูลตลาด + ให้ช่วงที่ปลายล่างคือเป้าจริง
Q3. 🔐 ในรอบ system design ทำไมการพูดถึง security ทำให้คุณโดดเด่น?
เฉลยเพราะ candidate ส่วนใหญ่โฟกัสแค่ functionality + scale แล้วลืม security · คุณ (ex-pentester) ใส่ threat model/auth/rate-limit เข้าไปธรรมชาติ = แสดงมุมที่คนอื่นมองข้าม + ตอกย้ำ positioning secure full stack
Q4. เมื่อเล่า capstone ทำไมต้องเตรียม "จุดที่จะทำต่างออกไป"?
เฉลยเพราะ interviewer มักถามเพื่อวัดวุฒิภาวะ/การสะท้อนตัวเอง · ตอบได้อย่างซื่อสัตย์ (เช่น "จะเพิ่ม test E2E แต่แรก") แสดงว่าคุณเรียนรู้และไม่หลงตัวเอง — น่าเชื่อถือกว่าอ้างว่าสมบูรณ์แบบ

7 · 🧹 Clean Code: สื่อสารให้คมเหมือนโค้ดที่อ่านง่าย

❌ ตอบยืดยาว วกวน

เล่าทุกรายละเอียดไม่เรียงลำดับ · interviewer หลงทาง · ไม่รู้ประเด็นหลักคืออะไร · เหมือนฟังก์ชัน 200 บรรทัด

✅ ตอบมีโครง กระชับ

สรุปก่อน (headline) → ลงรายละเอียดถ้าถูกถาม · ใช้ STAR · หยุดให้ interviewer ถามต่อ · เหมือนฟังก์ชันที่ตั้งชื่อดี

เคล็ด: ตอบแบบ "headline ก่อน รายละเอียดตาม" (เหมือน abstraction ที่ดี) · ซ้อมจนเล่า capstone ได้ 2 นาที (สั้น) และ 5 นาที (ยาว) · อัดเสียงตัวเองฟัง = เจอจุดวกวน · พูดช้า ชัด มั่นใจ ดีกว่าพูดเร็วแล้วรก · ไม่รู้ก็บอก "ไม่เคยใช้ แต่จะเข้าหาแบบนี้" — ซื่อสัตย์ชนะเสมอ
