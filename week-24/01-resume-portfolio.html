Week 24 · บทที่ 1

📄 Resume + Portfolio ที่ตรงกับฝีมือจริง

เปลี่ยน 23 สัปดาห์ให้เป็นหลักฐานที่ recruiter อ่านแล้วอยากเรียกสัมภาษณ์

1 · Resume คือเครื่องมือ "ผ่านด่านแรก" ไม่ใช่ชีวประวัติ

recruiter ใช้เวลากับ resume แต่ละใบ ~6-10 วินาทีในรอบแรก · เป้าหมายของ resume ไม่ใช่เล่าทุกอย่างที่คุณทำ แต่คือ ทำให้คนอ่านเชื่อภายใน 10 วินาทีว่าคุณทำงานนี้ได้ แล้วเรียกสัมภาษณ์

โครง resume สาย dev ที่ได้ผล (1 หน้า สำหรับ 1-5 ปี):

  • หัว — ชื่อ, ตำแหน่งที่อยากได้ (Secure Full Stack Developer), GitHub, LinkedIn, อีเมล
  • Summary 2-3 บรรทัด — คุณคือใคร + จุดขายเฉพาะ (ex-pentester + full stack)
  • Skills — จัดกลุ่ม (Frontend / Backend / DevOps / Security) เฉพาะที่ใช้จริงได้
  • Projects — capstone + library-system + automation report (พร้อม link + ตัวเลข)
  • Experience — media-presto + Royal Thai Army (pentester) — bullet วัดผล
  • Education / Certs — + free certs ที่ทำ (PortSwigger, CTF)
ตอบโจทย์ที่ 2 ของคุณโดยตรง: "resume ตรงกับโค้ดที่ทำได้จริง" — ทุกบรรทัดมี link/ตัวเลขรองรับ เพื่อให้ในห้องสัมภาษณ์คุณป้องกันทุก bullet ได้ ไม่มีอะไรที่อธิบายไม่ได้

2 · Bullet แบบ XYZ — Proof > Claim

สูตรที่ Google แนะนำ: "ทำ [X] วัดผลได้ [Y] ด้วยวิธี [Z]" · เปลี่ยนคำคุยโม้เป็นข้อเท็จจริงที่พิสูจน์ได้

❌ Claim (อ้างลอย)

"พัฒนาระบบ web ด้วย React และ Node.js"
"ดูแลเรื่อง performance"
"ใส่ใจ security"

✅ XYZ (มีหลักฐาน)

"ลด response time ของ dashboard 850ms→45ms ด้วย Redis cache + DB index (วัดด้วย EXPLAIN)"
"พบและแก้ช่องโหว่ IDOR + 3 จุดจาก self-pentest ตาม OWASP"

ตัวอย่าง bullet จาก capstone คุณ (ใช้ได้จริง)

VulnTracker — Secure pentest tracking platform  [Live] [GitHub]
• ออกแบบระบบจากศูนย์: design doc + threat model (STRIDE),
  RBAC 3 ระดับ, audit log ครบทุก action
• พบ+แก้ IDOR & broken access control ผ่าน self-pentest (OWASP Top 10)
• ลด dashboard latency 850ms→45ms (Redis cache-aside + composite index)
• CI/CD: GitHub Actions (lint+test+gitleaks+Semgrep), Docker, deploy Railway
• Stack: TypeScript, Express, PostgreSQL, Next.js, Redis, BullMQ
⚠️ counter-example: อย่าใส่ตัวเลขปลอม · "เพิ่ม performance 300%" ที่อธิบายไม่ได้ = ตายในห้องสัมภาษณ์ · ตัวเลขทุกตัวต้องมาจาก PERFORMANCE.md จริงที่คุณวัดเอง · proof ที่แท้จริงเท่านั้น

3 · จุดขาย ex-pentester — ความได้เปรียบที่คนอื่นไม่มี

full stack dev มีเยอะ · full stack dev ที่คิดแบบ attacker ได้หายาก · นี่คือ moat ของคุณ — ต้องดันให้เด่น ไม่ใช่ซ่อนไว้ท้าย resume

สินทรัพย์ของคุณเขียนใน resume/สัมภาษณ์ยังไง
6 ปี pentester (Royal Thai Army)Summary บรรทัดแรก — "Full stack dev with 6 yrs offensive security background"
OWASP Top 10 + CTFSkills section + เล่าว่า apply กับ capstone จริงยังไง
self-pentest capstoneSECURITY-AUDIT.md ใน repo — โชว์ว่า build แบบ security-first
เป้า AI-Security Engineerทิศทางที่ชัด — บริษัทชอบคนที่รู้ว่าตัวเองไปไหน
positioning: คุณไม่ใช่ "junior full stack อีกคน" · คุณคือ "secure full stack ที่ pentest เป็น" — ตำแหน่งที่ตลาดไทยขาดแคลนและจ่ายแพง · นี่คือเส้นทางสู่ 100k และ AI-Security

4 · Portfolio / GitHub — ด่านที่ recruiter คลิกดูจริง

recruiter สาย tech คลิก GitHub คุณจริง · จัดให้ด่านแรกสวย:

  • Pin 3 repo เด่น — capstone, library-system, automation-report (มี README + screenshot + live URL)
  • Profile README — แนะนำตัวสั้น + จุดขาย + 3 project + tech
  • Commit history สะอาด — conventional commits, ไม่มี "fix" "asdf" รัว ๆ
  • Course repo (public) — แสดง 24 สัปดาห์ของการเรียนรู้อย่างเป็นระบบ (นี่ก็คือ portfolio)
  • vee-securedev.vercel.app — landing page รวม project + resume + contact
💡 เคล็ด: repo ที่ pin ต้อง clone-แล้ว-รันได้ใน 5 นาที (README + .env.example + docker compose up) · recruiter ที่รัน project คุณได้จริง = ความประทับใจที่ resume สวยแค่ไหนก็สู้ไม่ได้

5 · ภาพรวม + คำช่วยจำ

Resume: Summary (จุดขาย) Skills (จริง) Projects (link+ตัวเลข) Experience (XYZ)

สูตร bullet: ทำ X · วัดผล Y · ด้วยวิธี Z

"Proof > Claim · เขียนแค่ที่อธิบายได้ · ดันจุดขาย pentester ขึ้นหัว · GitHub ต้อง clone-รันได้"

6 · Quiz

Q1. ทำไม resume bullet ควรเป็น XYZ (มีตัวเลข) แทน "พัฒนาระบบด้วย React"?
เฉลยXYZ พิสูจน์ผลกระทบที่วัดได้ ("850→45ms ด้วย cache+index") ทำให้ recruiter เห็นคุณค่าและความลึก · claim ลอย ๆ ใครก็เขียนได้ ไม่แยกคุณออกจากคนอื่น · และ XYZ บังคับให้คุณเขียนแค่สิ่งที่ทำจริง
Q2. ทำไม "ความซื่อสัตย์ใน resume" เป็นกลยุทธ์ ไม่ใช่แค่จริยธรรม?
เฉลยเพราะทุก bullet จะถูกขุดในห้องสัมภาษณ์ ("เล่าให้ฟังหน่อย") · ถ้าเขียนเกินฝีมือ คุณตอบลึกไม่ได้ = เสียความน่าเชื่อถือทั้งใบ · resume ที่ป้องกันตัวเองได้ทุกบรรทัด = แข็งแกร่งกว่า resume ที่สวยแต่กลวง
Q3. 🔐 ทำไมพื้นเพ ex-pentester ควรอยู่ "หัว" resume ไม่ใช่ซ่อนท้าย?
เฉลยเพราะมันคือ differentiator ที่หายาก — full stack ที่ pentest เป็นมีน้อยและตลาดไทยขาดแคลน/จ่ายแพง · ดันขึ้นหัว = recruiter เห็นทันทีว่าคุณไม่ใช่ junior ทั่วไป · เป็น positioning สู่ secure full stack / AI-Security
Q4. ทำไม repo ที่ pin ควร "clone แล้วรันได้ใน 5 นาที"?
เฉลยrecruiter สาย tech คลิกดู GitHub จริง · repo ที่รันได้ทันที (README+env.example+docker compose) พิสูจน์ว่าทำงานได้จริงและคิดถึงคนอื่น · ความประทับใจนี้แรงกว่า resume สวย — เป็นหลักฐานเป็นรูปธรรม

7 · 🧹 Clean Code: resume ที่ "อ่านง่าย" เหมือนโค้ดที่ดี

❌ resume รก อ่านยาก

3 หน้า · ทุก skill ที่เคยแตะ · paragraph ยาว · ไม่มีลำดับความสำคัญ · recruiter หาจุดเด่นไม่เจอใน 10 วิ

✅ resume กระชับ มีลำดับ

1 หน้า · เด่นสุดอยู่บน · bullet สั้นวัดผล · เว้นวรรคหายใจ · skim ได้ใน 10 วิ เจอจุดขายทันที

เคล็ด: resume เหมือนโค้ดสะอาด — ตัดสิ่งไม่จำเป็น (skill ที่ไม่ใช้/งานเก่าไม่เกี่ยว), เรียงตามความสำคัญ (เด่นบน), consistent (รูปแบบวันที่/bullet เหมือนกัน) · export เป็น PDF (ฟอนต์ไม่เพี้ยน) · ตั้งชื่อไฟล์ Varat-Dube-Secure-FullStack.pdf ดูเป็นมืออาชีพ
