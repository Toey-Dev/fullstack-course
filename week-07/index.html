🧹 Clean Code ประจำ Week 7

1. "Make impossible states impossible" — ออกแบบ type ให้ผิดไม่ได้

❌ type หลวม (สถานะผิดเกิดได้)

type Res = {
  ok: boolean;
  data?: Book;
  error?: string;
};
// เกิด { ok: true, error: "x" } ได้ (ขัดแย้ง!)

✅ discriminated union (ผิดไม่ได้)

type Res =
  | { ok: true; data: Book }
  | { ok: false; error: string };
// ok:true ต้องมี data / ok:false ต้องมี error

2. ใช้ type guard แทน as (type assertion)

❌ as = โกหก TS

const book = data as Book;  // บังคับ ไม่ได้เช็คจริง
book.title;  // พังถ้า data ไม่ใช่ Book จริง

✅ เช็คจริงด้วย guard

if (isBook(data)) {
  data.title;  // TS มั่นใจว่า Book จริง
}
กฎทอง: ออกแบบ type ให้สถานะขัดแย้งเกิดไม่ได้ · เลี่ยง as (assertion = ปิดการตรวจ) · ใช้ discriminated union กับทุกอย่างที่มี "หลายสถานะ" (API response, state, event)
