Week 8 — Next.js + Tailwind + Full-Stack

ประกอบหน้าบ้านกับหลังบ้านเป็นระบบเต็ม — library-system มีหน้าตาที่ใช้งานได้จริง

🎯 เป้าหมายของสัปดาห์นี้

เฟส 1 ที่ผ่านมาสร้าง backend (API) แข็งแรง — Week 8 สร้าง frontend ที่ต่อกับมัน ด้วย Next.js (React framework ที่ตลาดไทยต้องการ) + Tailwind (utility CSS) · คุณจะได้ full-stack app ที่ login ได้ ดูหนังสือได้ ยืมได้ — เชื่อมทุกอย่างที่เรียนมา 8 สัปดาห์

  • Next.js App Router: routing, layout, server vs client components
  • Tailwind: utility-first (ต่อยอด Bootstrap ที่รู้แล้ว)
  • fetch จาก library-system API + จัดการ 4 สถานะ (loading/success/error/empty)
  • หน้า login + เก็บ token + protected page (ใช้ความรู้ auth Week 5)
  • full-stack app เชื่อม frontend ↔ backend ที่สร้างเอง
ทำไมสัปดาห์นี้ปิดเฟส 1 ได้สมบูรณ์: ตอนนี้คุณมีทั้งหลังบ้าน (API: TS/test/DB/auth) และหน้าบ้าน (Next.js) — full-stack จริง · Next.js เป็นที่ต้องการในตลาดไทยมากกว่า React เปล่า ๆ และเป็นคำที่ใส่ใน resume ได้เลย

🧹 Clean Code ประจำ Week 8

1. แยก component ตามหน้าที่ (ไม่ยัดทุกอย่างในไฟล์เดียว)

❌ page เดียว 300 บรรทัด

// page.tsx — fetch + state + UI ทั้งหมดปนกัน
export default function Page() {
  // 300 บรรทัด...
}

✅ แยก component + hook

// useBooks.ts — logic fetch
// BookCard.tsx — UI การ์ด
// page.tsx — ประกอบ (บาง)

2. type props ทุก component (TS ต่อจาก Week 6-7)

type BookCardProps = { book: Book; onBorrow: (id: number) => void };
function BookCard({ book, onBorrow }: BookCardProps) { ... }
กฎทอง: component ทำหน้าที่เดียว · type props เสมอ · แยก data fetching (hook) จาก UI (component) · 'use client' เฉพาะที่ต้อง interactive (state/event) — default เป็น server component
