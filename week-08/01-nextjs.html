Next.js — App Router, Server/Client, Routing

React framework ที่ตลาดไทยต้องการ — เข้าใจว่ามันเพิ่มอะไรจาก React เปล่า ๆ

🔑 คีย์เวิร์ดภาพจำประจำบท

📁 โฟลเดอร์ = route (file-based routing) 🖥 server component = render ที่ server (default) 🖱 'use client' = interactive (state/event) 🎁 layout = กรอบที่ห่อทุกหน้า

🧠 แนวคิด & หลักการของ Dev ประจำบทนี้

หลักการ 1: "Next.js = React + สิ่งที่ production ต้องการ"

React เปล่า ๆ ให้แค่ "วาด UI" — แต่เว็บจริงต้องการ routing, SEO, performance, server rendering อีกเยอะ Next.js เติมส่วนเหล่านี้ให้ครบ (file-based routing, SSR, image optimization) · นี่คือเหตุผลที่บริษัทไทย (LINE MAN, Agoda) ใช้ Next.js ไม่ใช่ React เปล่า — และทำไม resume ที่มี Next.js แข็งกว่า React อย่างเดียว

หลักการ 2: "Server-first — render ที่ server ก่อน ส่งของเสร็จไป"

App Router default ทุก component เป็น "server component" — render ที่ server แล้วส่ง HTML เสร็จไป browser (เร็ว, SEO ดี, ดึง data ใกล้ DB) · ใช้ 'use client' เฉพาะส่วนที่ต้อง interactive (ปุ่ม, form, state) หลักคิด: ทำที่ server ได้ทำที่ server — ส่งของน้อย เร็วกว่า

หลักการ 3: "Convention over configuration — โครงไฟล์คือ config"

Next.js ใช้ "ชื่อไฟล์/โฟลเดอร์" กำหนดพฤติกรรม — page.tsx = หน้า, layout.tsx = กรอบ, โฟลเดอร์ = path · ไม่ต้องเขียน config routing เอง · เหมือน Express ที่มี convention แต่ Next.js เคร่งกว่า (โครงไฟล์ = ระบบ)

ส่วนที่ 1: Setup + โครงสร้าง App Router

npx create-next-app@latest library-web   # เลือก TypeScript + Tailwind + App Router

src/app/
├── layout.tsx          # กรอบห่อทุกหน้า (nav, font)
├── page.tsx            # หน้าแรก "/"
├── books/
│   └── page.tsx        # "/books"
├── books/[id]/
│   └── page.tsx        # "/books/123" (dynamic)
├── login/
│   └── page.tsx        # "/login"
└── globals.css         # Tailwind
ไฟล์/โฟลเดอร์กลายเป็น
โฟลเดอร์ books/path /books
page.tsxหน้าที่แสดงจริง
[id]/dynamic route (/books/123 → params.id = "123")
layout.tsxกรอบที่ห่อ page (nav, footer ที่ทุกหน้าใช้)

เทียบ Week 2: Express route เขียนใน code · Next.js route = โครงไฟล์ — convention แบบเดียวกัน คนละวิธีประกาศ

ส่วนที่ 2: Server vs Client Component

// Server Component (default) — fetch data ตรง
async function BooksPage() {
  const res = await fetch("http://localhost:3000/api/books");  // ดึงที่ server
  const { data } = await res.json();
  return <ul>{data.map(b => <li key={b.id}>{b.title}</li>)}</ul>;
}

// Client Component — ต้องมี 'use client' ถึงใช้ state/event ได้
"use client";
import { useState } from "react";
function SearchBar() {
  const [q, setQ] = useState("");
  return <input value={q} onChange={e => setQ(e.target.value)} />;
}
กฎเลือก: default = server (fetch, แสดงผล) · ใส่ 'use client' เฉพาะส่วนที่ต้อง state/event/hook · pattern จริง: หน้าเป็น server component (fetch) แล้วใส่ client component เล็ก ๆ (ปุ่ม, form) ข้างใน — ส่ง JS ไป browser น้อยสุด

ส่วนที่ 3: Layout + Navigation

// app/layout.tsx — กรอบที่ห่อทุกหน้า
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="th">
      <body>
        <nav>...เมนูที่ทุกหน้าใช้...</nav>
        {children}        {/* หน้าแต่ละ page มาแทรกตรงนี้ */}
      </body>
    </html>
  );
}

// navigation — ใช้ Link (ไม่ reload หน้า ต่างจาก <a>)
import Link from "next/link";
<Link href="/books">ดูหนังสือ</Link>

<Link> = navigation แบบ SPA (ไม่ reload หน้าทั้งหมด เร็วกว่า <a>) · layout = DRY ของ UI (nav/footer เขียนครั้งเดียว)

ส่วนที่ 4: ตัวอย่างประยุกต์

🏢 งานวี: dashboard การตลาดที่ทำให้ลูกค้า — server component fetch ข้อมูลสรุป (เร็ว, SEO ถ้าต้องการ), client component สำหรับ filter/กราฟ interactive · Next.js เหมาะกับ dashboard มาก (mix server+client)

🌏 อุตสาหกรรม: Next.js เป็น framework frontend อันดับต้นในไทยและโลก (LINE MAN, Agoda บางทีม, startup ส่วนใหญ่) · เข้าใจ server/client component = เข้าใจ React ยุคใหม่ (RSC) ที่ interviewer ถามแน่นอน

📝 Quiz ท้ายบท

ข้อ 1: Next.js เพิ่มอะไรจาก React เปล่า ๆ ทำไมบริษัทใช้?

React ให้แค่วาด UI · Next.js เติม routing (file-based), server rendering (SEO+เร็ว), image optimization, API routes — สิ่งที่เว็บ production ต้องการครบ · ทำให้ resume Next.js แข็งกว่า React อย่างเดียว

ข้อ 2: server component กับ client component ต่างกันยังไง เลือกอันไหนเมื่อไหร่?

server (default) render ที่ server — fetch data, SEO ดี, เร็ว, แต่ใช้ state/event ไม่ได้ · client ('use client') render ที่ browser — ใช้ useState/onClick/form ได้ · เลือก: default server, ใส่ 'use client' เฉพาะส่วน interactive

ข้อ 3: ทำไม <Link> ดีกว่า <a> ใน Next.js?

<Link> = navigation แบบ SPA — เปลี่ยนหน้าโดยไม่ reload ทั้งหน้า (เร็ว, รักษา state) + prefetch หน้าถัดไปให้ · <a> reload ทั้งหน้า (ช้า, state หาย) · ใช้ Link สำหรับ navigation ภายในเว็บเสมอ

ข้อ 4: layout.tsx มีไว้ทำอะไร เกี่ยวกับ DRY ยังไง?

กรอบที่ห่อทุกหน้า (nav, footer, font) — เขียนครั้งเดียวใช้ทุกหน้า แทนที่จะ copy nav ลงทุก page · เป็น DRY ของ UI เหมือน middleware ที่ใช้ร่วม แต่ฝั่ง layout
