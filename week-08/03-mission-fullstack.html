ประกอบทุกอย่างที่เรียนมา 8 สัปดาห์เป็น full-stack app จริง — ปิดเฟส 1 อย่างสมบูรณ์

ทำไม Mission นี้คือมงกุฎของเฟส 1

หลังจบ Mission นี้คุณมี full-stack app เต็มระบบ: หน้าบ้าน Next.js (TS+Tailwind) ↔ หลังบ้าน Express API (TS+test+DB+auth) ที่สร้างเอง · login ได้ ดูหนังสือได้ ยืมได้ · นี่คือ portfolio piece ที่เล่า interview ได้ 30 นาที และพิสูจน์คำว่า "Full Stack Developer" ใน resume ด้วยของจริง

ต้องมี backend รันอยู่: library-system API จาก Week 1-7 ต้องรัน (docker compose up + npm run dev) เพื่อให้ frontend ต่อได้

🪜 แบ่งเป็น 4 Mission ย่อย

Mission 8.1 — Setup + หน้า book list 75 นาที

  1. create-next-app (TS + Tailwind + App Router) ชื่อ library-web
  2. /books — fetch จาก API + แสดงการ์ด (Tailwind) responsive
  3. จัดการ 4 สถานะ (loading/success/error/empty) ด้วย FetchState union
✅ เทสต์: เปิด /books เห็นหนังสือจาก DB จริง · ปิด backend → เห็น error state ไม่ใช่จอขาว

Mission 8.2 — แยก component + type props 45 นาที

  1. แยก BookCard (UI) + useBooks hook (fetch logic)
  2. type props ทุก component (Book type จาก backend)
  3. page.tsx บาง (แค่ประกอบ)
✅ เทสต์: component แยกหน้าที่ชัด · type ครบ · build ผ่าน

Mission 8.3 — Login + Auth proxy 75 นาที

  1. /login — form ส่ง credential
  2. route handler /api/auth/login — คุย backend, เก็บ token ใน HttpOnly cookie
  3. /books/[id] — ปุ่มยืม (ต้อง login) → เรียกผ่าน proxy ที่แนบ token
security checklist: token ไม่อยู่ใน localStorage/ไม่โผล่ console · credential ส่งผ่าน POST · login error ตอบกลาง ๆ

Mission 8.4 — Deploy + ปิดเฟส 1 45 นาที

  1. deploy frontend ขึ้น Vercel (เชื่อม env ชี้ backend)
  2. README: screenshot + วิธีรัน (backend + frontend) + Tech Stack เต็ม + AI Usage
  3. push + pin repo — portfolio full-stack ชิ้นเอก
✅ เทสต์: เปิด URL deploy → ใช้งานได้จริง (ดูหนังสือ, login) จากมือถือ
